# Roola

Mac 向けの汎用ターミナルランチャーアプリ（Flutter Desktop / macOS）。
エクスプローラ機能をメインに、設定済みのディレクトリと動作（素のシェル / 任意コマンド / Claude Code Skill）をアイコンとして登録し、ワンクリックで起動できます。

## 主な機能

- **ファイルエクスプローラ**: メイン UI。ローカルディレクトリをブラウズし、ダブルクリックで遷移、右クリックでアクションメニュー。`.claude/skills/` を検知して特別アイコン化（Claude Code がインストールされている時のみ）
- **ランチャー（サイドバー）**: 登録済みエントリをサイドバーに並べ、ワンクリックで対象ディレクトリ + 動作を起動。フォルダで 1 階層グルーピング可
- **アプリ内 PTY ターミナル**: `xterm` ベースのフルターミナル。承認プロンプト・矢印キー操作・ANSI 制御に対応
- **3 種類の動作タイプ**:
  - **シェルを開く**: 対象ディレクトリで `$SHELL` をそのまま起動
  - **コマンドを実行**: 任意コマンドを `$SHELL -ilc` 経由で実行（完了後にシェル継続するかどうかも選択可能）
  - **Claude Skill 実行**: `claude /<skill-name>` を起動（Claude Code CLI がインストールされている場合のみ）
- **複数セッションの並行実行**: サイドバーのセッション chip 列で実行中・終了済みセッションを管理。アイコンには状態バッジを表示
- **外観カスタマイズ**: 透過 / 単色 / 画像 / グラデーション の 4 モードを切替可能。タイル表示密度も切替可能
- **マルチウィンドウ**: メニューバー「ファイル → 新規ウィンドウ」または Dock 右クリックから別プロセスの Roola を起動

## 前提条件

- macOS（Apple Silicon / Intel）
- Flutter SDK（バージョンは `.fvmrc` 参照 / [FVM](https://fvm.app/) 推奨）

[Claude Code](https://docs.claude.com/claude-code) CLI はオプションです。インストールされていない環境でも Roola はファイルエクスプローラ + シェル / コマンドランチャーとして動作し、Claude 関連 UI（Skill アイコン・右クリックメニューの「Claude Code を開く」「Skill 実行」「Skill をランチャーに登録」・Claude Skill 動作タイプ）だけが自動的に隠れます（ADR-0022）。

## インストール

### 配布版を使う（推奨）

公証 + ステープリング済みの DMG を使えば、Mac の標準的な手順でインストールできます。`build/Roola.dmg` を開いて Roola.app を `Applications` フォルダにドラッグしてください。Gatekeeper のチェックは事前に通っているため、ダブルクリックで普通に起動します。

### ソースからビルド

```bash
# 初回セットアップ（pub get + build_runner）
make setup

# 起動（開発ビルド）
make run

# 配布用 DMG をビルド（要: Developer ID Application 証明書 + notarytool プロファイル）
make dist
```

`make` を使わない場合の同等コマンド:

```bash
flutter pub get
dart run build_runner build
flutter run -d macos --dart-define-from-file=dart_defines/prod.json
```

FVM 利用時は `make run FLUTTER="fvm flutter" DART="fvm dart"` のように上書きできます。`make help` で全ターゲットを確認できます。

## 使い方

### 1. 初回起動

アプリを起動するとエクスプローラ画面が開きます。サイドバーには「お気に入り」と「ランチャー」のセクションがあり、未登録の状態ではランチャーは空です。

### 2. ランチャーにエントリを登録する

サイドバーの「ランチャー」セクション右端の「+」ボタン、またはサイドバーの「管理…」リンクから登録画面を開けます。動作タイプは 3 種類から選択します：

| 動作タイプ | 用途 |
|---|---|
| **シェルを開く** | `$SHELL` を対象ディレクトリで起動（コマンドなし） |
| **コマンドを実行** | 任意コマンドを `$SHELL -ilc` 経由で実行。`.zshrc` の PATH を継承するので pnpm / nvm / Homebrew 配下のツールもそのまま動く。完了後にシェルを継続させるオプションあり |
| **Claude Skill 実行** | `claude /<skill-name>` を起動。Skill 名を指定。Claude Code CLI がインストールされていないと選択肢自体が出ません |

入力項目:

| 項目 | 内容 |
|---|---|
| 表示名 | サイドバーに表示される任意のラベル（例: `my-app の dev サーバ`） |
| ディレクトリ | 起動時の作業ディレクトリの絶対パス。右側のフォルダアイコンでピッカーから選択 |
| 動作タイプ | 上記 3 種類のいずれか |
| コマンド / Skill 名 | 動作タイプにより必須項目が変わります |

「保存」を押すと永続化され、サイドバーに即時反映されます。

#### フォルダで整理する

登録済みエントリは 1 階層のフォルダでグルーピングできます。ランチャー管理画面でドラッグ & ドロップでフォルダに入れたり並べ替えたりできます。

### 3. エントリを起動する

サイドバーのエントリをクリックすると、PTY セッションが立ち上がり、画面右側に xterm ベースのターミナルが表示されます。状態は以下のいずれか：

- `起動中…` → `実行中` → `入力待ち` → `完了 (0)` / `失敗` / `キャンセル`

操作ボタン:

- **⏹ キャンセル**（実行中）: PTY を SIGTERM。出力履歴と終了状態のセッションは残り、再実行可能
- **🔄 再実行**（終了後）: 同じエントリで新しいセッションを起動
- **✕ 閉じる**（終了後）: セッションを完全に破棄

### 4. 複数セッションの並行実行

異なるエントリのアイコンを順次クリックすれば複数セッションを並行に動かせます。同じエントリは 1 セッションまで（再クリックは既存セッションに復帰）。サイドバー上部の chip 列で全セッションを一覧でき、chip タップで切替、chip 右の ✕ で破棄できます。**「閉じる」を押すまでセッションは残り続けます**ので、終了済みの出力履歴も保持されます。

ad-hoc 起動（後述のエクスプローラ右クリック経由）でも、`my-app` のような Finder 連番方式（同じ表示名が複数あれば ` 2`, ` 3` と振る）で並行実行できます。

### 5. アプリ終了時の確認

セッションが 1 件でも残っている状態でウィンドウを閉じる操作（× ボタン / Cmd+Q）を行うと、確認ダイアログ「N 件のセッションが残っています。終了するとすべての PTY が終了され、出力履歴も失われます」が出ます。「終了する」で全 PTY を SIGTERM して終了、「キャンセル」で継続。セッションが残っていなければ確認なしで閉じます。

### 6. ターミナル操作

通常のターミナルと同じ感覚で操作できます:

- **テキスト入力 + Enter**: `\n` 付きで PTY に書き込み
- **承認応答**: `y` / `n` 等で `claude` などの確認プロンプトに応答
- **矢印キー (↑↓←→)**: ANSI エスケープシーケンスで PTY に送信。選択 UI を操作可能
- **Ctrl-C**: 0x03 を送信。`claude` 側で SIGINT 相当の中断
- **ウィンドウリサイズ**: PTY 側の cols/rows も自動追従

### 7. エクスプローラ

メイン画面そのものがエクスプローラです。

- **ルートディレクトリ**: 初回はホームディレクトリから。AppBar の「📂↑」ボタンで別ディレクトリに変更でき、選択した場所は次回起動時にも記憶されます
- **ナビゲーション**: ダブルクリックでサブディレクトリに遷移（ADR-0021）。AppBar 左上の戻る矢印で 1 階層上に移動
- **シングルクリック**: 選択（パスバーに反映）
- **Cmd+C**: 選択中アイテムの絶対パスをクリップボードにコピー
- **タイル表示密度**: 設定画面で「コンパクト / ゆったり」を切替可能（ADR-0024）
- **ファイル右クリック**: Finder で表示・OpenWith・名前変更・コピー・プロパティ等
- **フォルダ右クリック**: 上記に加えて、Claude Code がインストールされていれば以下も表示
  - 「このディレクトリで Claude Code を開く」: Skill 引数なしで `claude` 対話モードを起動。chip 列に「ディレクトリ名 (Claude)」で並ぶ
  - **「`<skill>` を即実行」**（`.claude/skills/` 検知時のみ）: エントリ登録なしでその場で Skill を起動 → chip 列に「ディレクトリ名 / Skill 名」で並ぶ
  - **「`<skill>` をランチャーに登録」**（同上）: ランチャー登録画面をディレクトリパス + Skill 名プリフィル状態で開く

「即実行」「Claude Code を開く」で起動した ad-hoc セッションはランチャーには永続追加されません。chip の ✕ で破棄するまで残ります。

### 8. 外観のカスタマイズ

設定画面の「外観」セクションで背景モードを切替できます:

- **透過**（デフォルト）: 半透明背景でデスクトップが透けて見える。透過率はスライダーで調整可能
- **単色**: プリセット色から選択
- **画像**: ローカル画像を選択し背景にカバー表示
- **グラデーション**: ロゴ準拠のグラデーション

設定変更は即時反映されます。

### 9. Claude Code のヘルスチェック

設定画面で `claude --version` の結果バナーが表示されます。Claude Code 未インストールの場合は「未検出」と表示され、関連 UI が自動的に隠れます。インストール手順は同画面の「インストールガイド」セクション参照。

GUI 起動経路（Dock / Finder）では launchd 由来の最小 PATH に切り詰められるため、Roola はヘルスチェックでも PTY 起動でも `$SHELL -ilc` 経由で claude を呼びます（`.zshrc` の PATH 拡張も継承）。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| ランチャーで `command not found` | `which <command>` でターミナルから動くか確認。動くのに Roola から動かない場合は `.zshrc` の PATH 設定を見直し（Roola は `$SHELL -ilc` 経由で実行するので `.zshrc` の設定が継承されます） |
| 「Claude Code が見つかりません」と表示される | `which claude` で確認。複数バージョンが PATH 上に共存していると古い方が選ばれることがあります |
| 「ディレクトリが見つかりません」とエラー | エントリ編集でディレクトリパスを正しい値に更新 |
| ターミナル描画が崩れる | ウィンドウサイズを変更すると再レイアウトされます |
| Mac スリープから復帰したら一部 Skill が `failed` になっている | スリープ中に Anthropic API への SSE ストリーミングがタイムアウトで切断されたため。承認待ち / 完了済みのセッションは影響なし。chip を「閉じる」で破棄して必要なら再実行してください |
| 永続化されたエントリ・設定をリセットしたい | `make reset`（prod / dev の Application Support 配下を両方削除） |

## 既知の制約

- macOS 専用（Windows / Linux サポートは予定なし）
- App Sandbox は **無効化** されています（PTY 起動と任意ディレクトリへのアクセスのため）
- `riverpod_lint` / `custom_lint` は riverpod 3.x との依存解決の都合で当面保留中（ADR-0007）
- 実 PTY 上での完了・キャンセル系のテストは `flutter test` のテストバインディングでは検証できず `integration_test` 対象

## 開発者向け情報

設計判断の時系列記録は `docs/adr/` を参照してください（ADR-0001 〜 ADR-0025）。アーキテクチャの詳細は `docs/architecture.md`、コーディング規約は `docs/coding-standards.md`。

仕様駆動開発として [OpenSpec](https://github.com/lukasvalle/openspec) を併用しています。新しい change は `openspec new change <name>` で起こせます。

開発時の典型コマンド:

```bash
make watch           # コード生成を watch モードで起動
make check           # format → analyze → test（CI 相当）
make dmg             # ビルド + 署名 + 圧縮 DMG 作成
make dist            # 上記 + Apple へ提出して公証 + ステープル
make reset           # 永続化されたエントリ・設定を削除
```

## Project status / Support

Roola は個人の週末プロジェクトとして開発されている OSS です。「自分が使うため」が
第一の目的で、コミュニティサポート義務は負いません。

- **Issue / PR の方針**:
  - issue は再現可能な最小ケース（[テンプレート](./.github/ISSUE_TEMPLATE/bug_report.md)
    に沿った報告）から優先的に扱います。要望系の issue にも反応しない可能性があります。
  - PR は歓迎しますが、メンテナの方針に合わないものは merge されないこともあります。
    大きい変更を送る前に、まず issue で方針を確認することをお勧めします。
  - CLA（Contributor License Agreement）は取りません。
- **ロードマップは公開しません**。次に何を作るかはメンテナの気分次第です。
- **後方互換性は最小限**: メジャーバージョン 0.x の間は破壊的変更が予告なく入ります。

商用サポートが必要な方は別途相談してください（連絡先は [GitHub プロフィール](https://github.com/yahir0) を参照）。
