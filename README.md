# Claude Skills Launcher

Mac 向けの Claude Code Skills ランチャーアプリ（Flutter Desktop / macOS）。
登録したリポジトリと Skill の組み合わせをアイコンとして並べ、ワンクリックで対象ディレクトリで Skill 実行まで到達することを目的にしています。

## 主な機能

- **アイコングリッド**: ホーム画面に登録済みエントリをアイコンで並べ、クリックで実行画面へ
- **アプリ内 PTY ターミナル**: `xterm` ベースのフルターミナルを内蔵。承認プロンプト・矢印キー操作・ANSI 制御に対応
- **設定画面**: エントリ追加・編集・削除、アイコン画像登録、リポジトリパスのディレクトリ選択
- **外観カスタマイズ**: 透過 / 単色 / 画像背景の切り替え
- **ヘルスチェック**: `claude` CLI の存在確認と設定画面でのバナー表示

## 前提条件

- macOS（Apple Silicon / Intel どちらも）
- [`claude`](https://docs.claude.com/claude-code) CLI が PATH に通っていること
- `git` が PATH に通っていること
- Flutter SDK（バージョンは `.fvmrc` 参照 / [FVM](https://fvm.app/) 推奨）

## セットアップと起動

```bash
# 初回セットアップ（pub get + build_runner）
make setup

# 起動
make run
```

`make` を使わない場合の同等コマンド:

```bash
flutter pub get
dart run build_runner build
flutter run -d macos --dart-define-from-file=dart_defines/prod.json
```

VSCode から起動する場合は `.vscode/launch.json` の「Launch (prod)」コンフィグを使ってください。

FVM 利用時は `make run FLUTTER="fvm flutter" DART="fvm dart"` のように上書きできます。`make help` で全ターゲットを確認できます。

## 使い方

### 1. 初回起動

アプリを初めて起動すると、ホーム画面に「登録されたランチャーがまだありません」とプレースホルダーが表示されます。「設定画面を開く」ボタン、または右上の歯車アイコンから設定画面に移動します。

### 2. エントリを追加する

設定画面右上の「＋」ボタンからエントリ追加画面を開き、以下を入力します:

| 項目 | 内容 |
|---|---|
| 表示名 | アイコン下に表示される任意のラベル（例: `my-app の lint 修正`） |
| リポジトリパス | Skill を実行するローカルリポジトリの絶対パス。右側のフォルダアイコンからディレクトリピッカーで選択できます |
| Skill 名 | `claude` に引数として渡す Skill 名（例: `tdd-red`、`commit` など） |
| アイコン | 任意の画像ファイル。選択すると 512px の正方形 PNG にリサイズしてアプリのサポートディレクトリへ保存します。未指定の場合はデフォルトアイコンが使われます |

「保存」ボタンを押すと永続化され、ホーム画面のグリッドに即時反映されます。

**バリデーション:**

- 表示名・Skill 名が空 → エラーメッセージを TextField の下に表示
- リポジトリパスが存在しない → 同様にエラー
- どれか 1 つでもエラーがあると保存できません

### 3. Skill を実行する

ホーム画面のアイコンをクリックすると実行画面に遷移し、自動的に対象ディレクトリで `claude` プロセスを PTY 上で起動します。

実行画面の構成:

- **タイトル**: エントリの表示名
- **状態バッジ**: `待機中` / `起動中…` / `実行中` / `完了 (0)` / `失敗` / `キャンセル` のラベルとカラー
- **操作ボタン**: 実行中は「⏹ キャンセル」、終了後は「🔄 再実行」、いつでも「🏠 ホームへ戻る」
- **ターミナルビュー**: xterm ベース。Claude Code が出す承認プロンプト・矢印キー選択 UI・ANSI 制御を解釈して描画します

### 4. ターミナル操作

PTY 統合のため、通常のターミナルと同じ感覚で操作できます:

- **テキスト入力 + Enter**: 改行 (`\n`) 付きで PTY に書き込まれる
- **承認応答**: `y` や `n` を入力 → そのまま `claude` の確認プロンプトに応答
- **矢印キー (↑↓←→)**: ANSI エスケープシーケンスとして PTY に送信。Claude Code の選択 UI を操作できます
- **Ctrl-C**: 0x03 を送信し、`claude` 側で SIGINT 相当の中断
- **ウィンドウリサイズ**: アプリウィンドウを変えると PTY 側の cols/rows も自動追従

### 5. エントリの編集・削除

設定画面に並んだエントリ行をタップすると編集画面が開き、既存値がプリロードされます。同じフォームで上書き保存できます。

行右端の 🗑 削除ボタンを押すと確認ダイアログが出て、「削除」を選ぶと永続化ファイルとアイコン画像から取り除かれます。

### 6. 背景・外観のカスタマイズ

設定画面上部の「外観」セクションで背景モードを切り替えられます:

- **透過**（デフォルト）: ウィンドウ背景を透明にし、デスクトップが透けて見える
- **単色**: プリセット 6 色から選択
- **画像**: ローカルの画像ファイルを選び、`<appSupport>/background.png` にコピー保存して背景にカバー表示

設定変更は即時反映され、再起動は不要です。

### 7. `claude` コマンドのヘルスチェック

設定画面上部に `claude --version` を実行した結果のバナーが出ます。コマンドが見つからない場合は赤バナーで警告。インストールと PATH を確認してください。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| 「`claude` コマンドが見つかりません」と表示される | `which claude` で PATH を確認。`~/.zshrc` などにパスを追加した後、アプリを再起動 |
| 「リポジトリディレクトリが見つかりません」と実行画面でエラー | 設定画面で該当エントリを編集し、リポジトリパスを正しい値に更新 |
| ターミナルの描画が崩れる | ウィンドウサイズを一度変更すると再レイアウトされます |
| アプリ自体が起動しない | `flutter clean && flutter pub get && dart run build_runner build` の順で再ビルド |
| 永続化されたエントリ・設定をリセットしたい | `~/Library/Application Support/io.github.yahir0.claude_skills_launcher/` 配下のファイルを手で削除 |

## 既知の制約

- macOS 専用です（Windows / Linux サポートは現時点では行っていません）
- App Sandbox は **無効化** されています（PTY 起動と任意ディレクトリへのアクセスのため）。配布フェーズで再評価が必要です
- `riverpod_lint` / `custom_lint` は riverpod 3.x との依存解決の都合で現在保留中（詳細は `docs/adr/0007-riverpod-lint-deferral.md` を参照）
- 実 PTY 上での完了・キャンセル系のテストは `flutter test` のテストバインディングでは検証できず、`integration_test` 対象です

