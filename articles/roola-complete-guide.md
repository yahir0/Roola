## Roola とは

**Roola** は「目的地へ、一瞬で」をコンセプトにした、開発者向けの macOS ファイルエクスプローラです。ローカルのファイルをブラウズしながら、

- シェルを開く
- 任意のコマンドを実行する
- Claude Code の Skill を起動する

といった「動作」を、**ディレクトリ単位** にひもづけてワンクリックで実行できます。よく使う「ディレクトリ + 動作」のペアを一度登録しておけば、サイドバーからいつでも呼び出せます。実行はすべてアプリ内蔵の PTY ターミナルの中で走ります。

機能ハイライトやダウンロードリンクは公式サイトにもまとまっています。

https://yahiro.tech/roola

![Roola のワークスペース全体。左にサイドバー、中央に 3 ペインのタブ式ワークスペース、上部にトップバー（アクティビティモニタ・ノートパッド・設定）が見えるスクリーンショット](/images/roola-complete-guide/placeholder-hero.png)
*▲ スクショ：起動直後のワークスペース全体。サイドバー・3 ペイン・トップバーが一望できる画面*

:::message
**動作環境**
- macOS 14（Sonoma）以降を推奨。Apple Silicon / Intel の両方に対応
- Claude Code CLI は **オプション**（無くても汎用ランチャーとして完全に動作します）
:::

## インストール

1. [Releases](https://github.com/yahir0/Roola/releases) ページから最新の DMG をダウンロードします（署名 + 公証 + ステープリング済み）。
2. DMG を開いて `Roola.app` を `Applications` フォルダにドラッグします。
3. Launchpad または Spotlight から `Roola` で起動します。

自動アップデートは [Sparkle](https://sparkle-project.org/) が担当します。手動でも、メニューバーの「Roola → アップデートを確認…」からチェックできます。

![DMG を開いて Roola.app を Applications フォルダにドラッグするインストール画面](/images/roola-complete-guide/placeholder-install.png)
*▲ スクショ：DMG マウント時のインストール画面（Roola.app → Applications へのドラッグ）*

## 画面構成の全体像

起動すると Explorer タブが開いた状態でワークスペースが立ち上がります。画面は大きく 3 つの領域に分かれています。

| 領域 | 内容 |
|---|---|
| **トップバー** | ウィンドウのドラッグ領域。右側にアクティビティモニタ・ノートパッド・設定ボタン |
| **サイドバー** | PLACES / お気に入り / ランチャー / 実行中セッションの 4 セクション |
| **3 ペインワークスペース** | 左上・右上・下の 3 ペインに Explorer / Terminal / Git タブを自由配置 |

# 機能解説

## 1. 3 ペインタブ式ワークスペース

Roola のワークスペースは **左上・右上・下** の 3 つのペインに分かれています。それぞれのペインに Explorer / Terminal / Git の 3 種類のタブを自由に配置できます。

![3 ペイン（左上に Explorer、右上にもう 1 つの Explorer、下に Terminal）にタブが配置されたワークスペース。ペインの境界線が見える状態](/images/roola-complete-guide/placeholder-workspace.png)
*▲ スクショ：3 ペインそれぞれにタブが配置された状態（境界のスプリッタが見えるように）*

### タブの操作

| 操作 | ショートカット / マウス |
|---|---|
| 新規エクスプローラタブ | `⌘T` |
| 新規ターミナルタブ | `⌘⇧T` |
| タブを閉じる | `⌘W` |
| 次のタブ / 前のタブ | `⌃Tab` / `⌃⇧Tab` |
| タブを左上ペインへ移動 | `⌘⌃1` |
| タブを右上ペインへ移動 | `⌘⌃2` |
| タブを下ペインへ移動 | `⌘⌃3` |
| タブのドラッグ移動 | タブをドラッグして他ペインへ |
| ペインのリサイズ | 境界線（スプリッタ）をドラッグ |

:::message
**ワークスペースのレイアウトは永続化されません**。タブ構成・分割比率・各タブのカレントパスは保存されず、アプリを再起動すると毎回 **既定レイアウト**（Explorer ×2 ＋ ターミナル ×1 の 3 ペイン）から始まります。「閉じたら閉じる、開き直したらまっさら」という設計です。前回いた深い階層に戻りたいときは **お気に入り** を活用してください。
:::

## 2. ファイルエクスプローラ

ローカルのディレクトリをブラウズする中核機能です。

![エクスプローラタブ。ファイル・フォルダがタイル表示され、上部にパスバー（アドレスバー）、左に戻る/進むボタンがある画面](/images/roola-complete-guide/placeholder-explorer.png)
*▲ スクショ：エクスプローラタブ（ファイル一覧・パスバー・戻る/進むボタンが見える状態）*

### ナビゲーション

- **ダブルクリック**：フォルダに遷移／ファイルは OS の既定アプリで開く
- **シングルクリック**：選択（パスバーに反映）
- **パスバー**：アドレスバーに直接パスを入力して `Enter` で移動
- **戻る / 進む**：ペインヘッダのボタン、または `⌘[` / `⌘]`。マウスのサイドボタンにも対応
- **上の階層へ**：`⌘↑`

### キーボード操作

一覧を一度クリックしてフォーカスを得てから、十字キーで操作できます。

| キー | 動作 |
|---|---|
| `↑` / `↓` | 選択を移動 |
| `Enter` | 開く（ディレクトリは遷移、ファイルは既定アプリ） |
| `→` | ディレクトリに侵入 |
| `←` | 親ディレクトリへ戻る |

### ファイル操作（右クリックメニュー / ショートカット）

| 操作 | ショートカット |
|---|---|
| パスをコピー | `⌘⇧C` |
| コピー | `⌘⌥C` |
| ペースト | `⌘⌥V` |
| 名前を変更 | `⌘R` |
| 削除（ゴミ箱へ） | `⌘⌫` |
| 新規フォルダ | `⌘⇧N` |
| 新規テキストファイル | `⌘⌥N` |
| Finder で表示 | `⌘⌥R` |
| 別のアプリケーションで開く | `⌘O` |
| プロパティ | `⌘I` |

*▲ スクショ：右クリックのコンテキストメニュー（「このディレクトリで操作」セクションまで含めて）*

:::details プロパティダイアログに表示される項目
名前 / パス / 種類（ディレクトリ・ファイル）/ サイズ / 更新日時 / アクセス日時 / 変更日時 / パーミッション

### タイル表示密度

設定画面で **コンパクト / ゆったり** を切り替えられます。「ゆったり」では Skill のサブタイトルやチップも表示されます。

### サイドバー

サイドバーは 4 つのセクションで構成されています。

| セクション | 内容 |
|---|---|
| **PLACES** | Home / Downloads / Desktop / Documents / Applications と「別のフォルダを開く…」 |
| **お気に入り** | 任意のディレクトリを登録。`+` で現在地を登録／新規フォルダ作成。ドラッグで並べ替え |
| **ランチャー** | 登録した「ディレクトリ + 動作」エントリ。フォルダで 1 階層グループ化可。「管理…」で管理画面へ |
| **実行中（Running）** | 実行中のターミナルセッション一覧。クリックでそのタブへ、`×` で完全破棄 |

## 3. ファイルプレビューパネル

エクスプローラでファイルを選択すると、右ペインに **読み取り専用** のプレビューが開きます。

![ファイルプレビューパネル。左にファイル一覧、右にシンタックスハイライトされたソースコードのプレビューが表示された状態](/images/roola-complete-guide/placeholder-preview.png)
*▲ スクショ：ファイルプレビューパネル（コードのシンタックスハイライト表示）*

- **対応形式**：コード / テキスト（シンタックスハイライト付き）、画像（PNG / JPEG / GIF / WebP / BMP）、PDF
- **自動開閉**：既定は非表示。プレビュー可能なファイルを選ぶと自動で開き、ディレクトリやバイナリ・大きすぎるファイルを選ぶと自動で閉じる
- **操作**：画像 / PDF はパン・ズーム可。テキストは選択・コピーのみ（編集したいときは右クリック →「別のアプリケーションで開く」）
- **更新**：パネル右上のトグルで開閉、リフレッシュボタンで再読込

:::message
表示状態・幅はタブごとに保持されますが、これは永続化されません（再起動でリセットされます）。
:::

![画像ファイルを選択したときのプレビュー（パン・ズーム可能な画像表示）](/images/roola-complete-guide/placeholder-preview-image.png)

## 4. ワンクリック・ランチャー

Roola の主役機能です。「**ディレクトリ + 動作**」のペアを登録し、サイドバーからワンクリックで起動します。

### エントリの登録

サイドバーの「ランチャー」セクション右端の `+`、または「管理…」リンクから登録画面を開きます。動作タイプは 3 種類です。

| 動作タイプ | 用途 |
|---|---|
| **シェルを開く** | `$SHELL` を対象ディレクトリで起動（コマンドなし） |
| **コマンドを実行** | 任意コマンドを `$SHELL -ilc` 経由で実行。`.zshrc` の PATH を継承するので pnpm / nvm / Homebrew 配下のツールもそのまま動く。完了後にシェルを継続させるオプションあり |
| **Claude Code Skill** | `claude /<skill-name>` を起動。Claude Code CLI 未インストール時は選択肢が出ません |

![ランチャーエントリの編集画面。表示名・作業ディレクトリ・フォルダ・動作タイプの選択肢が見える状態](/images/roola-complete-guide/placeholder-launcher-edit.png)
*▲ スクショ：エントリ編集画面（表示名・作業ディレクトリ・動作タイプの 3 択が見える状態）*

エントリ編集画面の主な入力項目：

- **表示名**：サイドバーに表示される名前
- **作業ディレクトリ**：実行時のカレントディレクトリ（ディレクトリピッカー付き）
- **フォルダ**：グルーピング用（1 階層・未分類 or 任意フォルダ）
- **動作**：3 種類から選択。「コマンドを実行」ではコマンド文字列と「終了後もシェル継続」オプション、「Claude Code Skill」では Skill 名を指定

### エントリを起動する

サイドバーのエントリをクリックすると PTY セッションが立ち上がり、ワークスペースのターミナルタブに表示されます。セッションは「閉じる」を押すまで残り、終了後の出力履歴も保持されます。

### ランチャー管理画面

`⌘L`、またはメニューバーの「File → ランチャー管理を開く」で開きます。エントリ・フォルダの追加、ドラッグ & ドロップでの整理、削除ができます。

![ランチャー管理画面。フォルダとエントリの一覧、ドラッグで並べ替えできる UI](/images/roola-complete-guide/placeholder-launcher-manage.png)
*▲ スクショ：ランチャー管理画面（フォルダ・エントリ一覧と並べ替え UI）*

## 5. 内蔵 PTY ターミナル

ターミナルは [SwiftTerm](https://github.com/migueldeicaza/SwiftTerm) のネイティブビューで描画されており、高速かつ CJK 表示に強いのが特徴です（フォントは Sarasa Term J を同梱）。

![ターミナルタブ。コマンド実行中の出力が表示された状態](/images/roola-complete-guide/placeholder-terminal.png)
*▲ スクショ：ターミナルタブでのコマンド実行画面*

通常のターミナルと同じ感覚で操作できます。

- **テキスト入力 + Enter**：`\n` 付きで PTY に書き込み
- **Shift+Enter**：LF 改行をそのまま入力（Claude Code などの複数行入力用）
- **承認応答**：`y` / `n` で確認プロンプトに応答
- **矢印キー / Ctrl-C / ウィンドウリサイズ**：ANSI 制御に対応し、PTY 側の cols/rows も自動追従

:::message
GUI 起動経路（Dock / Finder）では launchd 由来の最小 PATH に切り詰められます。Roola はそれを避けるため、ヘルスチェックでも PTY 起動でも `$SHELL -ilc` 経由でコマンドを呼び、`.zshrc` の PATH 拡張を継承します。
:::

## 6. Git ビュー

Git 管理下のディレクトリで「Git ビューを開く」を選ぶと、新しいタブとして Git ビューが開きます。`git` CLI のラッパーとして実装されています。

![Git ビュータブ。Changes セクションで変更ファイル一覧と diff が表示された状態](/images/roola-complete-guide/placeholder-git.png)
*▲ スクショ：Git ビュー（Changes セクション：変更ファイル一覧と diff）*

Git ビューは 4 つのセクションで構成されています。

| セクション | 機能 |
|---|---|
| **Changes** | working tree の変更ファイル一覧と diff。Stage / Unstage、コミット、Amend |
| **History** | コミットグラフと、選択コミットの詳細・ファイル diff |
| **Branches** | ローカル / リモートブランチの一覧・作成・切替・マージ・削除、stash の退避と適用 |
| **Actions** | フェッチ / プル / プッシュ（Force Push は --force-with-lease） |

Git ビューにフォーカスがあるときは、以下のショートカットが使えます。

| 操作 | ショートカット |
|---|---|
| Git ビューを更新 | `⌘⇧R` |
| フェッチ | `⌘⇧F` |
| プル | `⌘⇧L` |
| プッシュ | `⌘⇧U` |

![Git ビューの History セクション。コミットグラフと選択コミットの詳細が表示された状態](/images/roola-complete-guide/placeholder-git-history.png)
*▲ スクショ：Git ビューの History セクション（コミットグラフ）*

:::details stash の操作
変更を stash に退避（メッセージ任意）、stash 一覧の表示、選択した stash の適用 / pop / 破棄ができます。
:::

## 10. キーボードショートカット

すべての主要操作はコマンドとして定義され、ネイティブメニューバーと統一されています。設定画面の「キーボードショートカット」リンク、または `⌘⌥,` で一覧・編集画面を開けます。

![キーボードショートカットの編集画面。コマンドごとに割り当てキーが表示され、各行をクリックすると記録ダイアログが開く画面](/images/roola-complete-guide/placeholder-keybindings.png)
*▲ スクショ：キーボードショートカット編集画面（コマンド一覧と割り当てキー）*

各行をクリックするとキーコンビ記録ダイアログが開き、修飾キーを含む新しいキーに割り当てできます。

:::message alert
`⌘C` / `⌘V` / `⌘X` / `⌘A` / `⌘Z` はテキスト編集用に予約されており、コマンドへの割り当てはできません。
:::

### タスク完了通知

Roola で起動した Claude Code がターンを完了して入力待ちに戻ると、macOS のローカル通知でお知らせします。長い処理の待ち時間に、別作業へ集中できます。

![Claude Code のタスクが完了したときに表示される macOS のローカル通知バナー](/images/roola-complete-guide/placeholder-notification.png)
*▲ スクショ：タスク完了時の macOS 通知バナー*

完了検知には Claude Code の **Stop フック** を使うため、`Esc` で中断したときは通知は出ません。検知対象は **Roola が起動した Claude Code セッションのみ** です。起動時にタブ ID + アプリ起動ごとのランダムトークンを環境変数として注入し、それを照合するので、Roola 外の claude や他プロセスからの偽通知は届きません。受信口は `127.0.0.1` のみに待ち受けます。

**セットアップ手順：**

1. 設定画面の「タスク完了通知」を **オン** にします。初回は macOS の通知許可を求められます。
2. 同じ画面に出る Stop フックのスニペット（待受ポート埋め込み済み）を `~/.claude/settings.json` の `hooks` に追記します。

:::message alert
既存の設定がある場合は、新しいオブジェクトを足すのではなく、**既存の `{ }` の中に `hooks` をマージ** してください（JSON のルートは 1 つだけです）。
:::

3. スニペットは `jq` と `curl` を使います（多くの環境に標準で入っています）。

![設定画面のタスク完了通知セクション。オン/オフトグル、macOS 許可状態、待受ポート、Stop フックのスニペットとコピーボタンが見える状態](/images/roola-complete-guide/placeholder-notification-setup.png)
*▲ スクショ：タスク完了通知のセットアップ画面（トグル・ポート・Stop フックスニペット）*

トークンは環境変数 `$ROOLA_NOTIFY_TOKEN` を参照するため、Roola を再起動しても貼り直しは不要です（待受ポートが変わったときだけスニペットを取り直してください）。

# トラブルシューティング

| 症状 | 対処 |
|---|---|
| ランチャーで `command not found` | `which <command>` でターミナルから動くか確認。動くのに Roola から動かない場合は `.zshrc` の PATH 設定を見直す（Roola は `$SHELL -ilc` 経由で実行するので `.zshrc` の設定が継承されます） |
| 「Claude Code が見つかりません」と表示される | `which claude` で確認。複数バージョンが PATH 上に共存していると古い方が選ばれることがあります |
| 「ディレクトリが見つかりません」とエラー | エントリ編集でディレクトリパスを正しい値に更新 |
| ターミナル描画が崩れる | ウィンドウサイズを変更すると再レイアウトされます |
| スリープ復帰後に一部 Skill が `failed` になっている | スリープ中に API への SSE ストリーミングがタイムアウトで切断されたため。chip を「閉じる」で破棄して必要なら再実行 |
| タスク完了通知が出ない | ①設定がオンで macOS 通知が許可されているか ②`~/.claude/settings.json` が壊れていないか（`python3 -m json.tool ~/.claude/settings.json` で検証）③スニペットのポートが設定画面の待受ポートと一致しているか ④`jq` / `curl` が PATH 上にあるか |

# 既知の制約

- macOS 専用（Windows / Linux サポートは予定なし）
- App Sandbox は無効化されています（PTY 起動と任意ディレクトリへのアクセスのため）
- メジャーバージョン 0.x の間は破壊的変更が予告なく入ることがあります
