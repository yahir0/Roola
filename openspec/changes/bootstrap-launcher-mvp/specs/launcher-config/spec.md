## ADDED Requirements

### Requirement: ランチャーエントリの永続化

システムは、ランチャーエントリ（一意 ID・表示名・リポジトリ絶対パス・実行する Skill 名・アイコン画像パス）の集合をローカルストレージへ永続化し、アプリ起動時に復元 SHALL する。

#### Scenario: 起動時にエントリを復元する

- **WHEN** ユーザーがアプリを再起動する
- **THEN** 前回保存したランチャーエントリ一覧がホーム画面に表示される

#### Scenario: 保存先が破損または存在しない

- **WHEN** 永続化ファイルが存在しない、または JSON として不正な状態である
- **THEN** システムは空のエントリ一覧として扱い、ユーザーへエラーを表示せず正常起動する

### Requirement: ランチャーエントリの追加

システムは、ユーザーが設定画面から新しいランチャーエントリを追加できるようにする SHALL。

#### Scenario: 必須項目を入力して追加する

- **WHEN** ユーザーが表示名・リポジトリパス・Skill 名・アイコン画像を入力して「保存」を押す
- **THEN** 新しいエントリが永続化され、ホーム画面に即時反映される

#### Scenario: リポジトリパスが存在しない

- **WHEN** ユーザーが入力したリポジトリパスがローカルに存在しない
- **THEN** システムは保存を拒否し、「指定されたディレクトリが見つかりません」のバリデーションエラーを表示する

#### Scenario: 表示名または Skill 名が空

- **WHEN** ユーザーが表示名または Skill 名を未入力のまま「保存」を押す
- **THEN** システムは保存を拒否し、該当フィールドに必須エラーを表示する

### Requirement: ランチャーエントリの編集

システムは、既存ランチャーエントリの全フィールドを編集できる SHALL。

#### Scenario: 既存エントリを編集して保存する

- **WHEN** ユーザーが設定画面のエントリ行をタップし、値を変更して保存する
- **THEN** 変更内容が永続化され、ホーム画面の表示も更新される

### Requirement: ランチャーエントリの削除

システムは、既存ランチャーエントリを削除できる SHALL。

#### Scenario: 削除確認を経て削除する

- **WHEN** ユーザーが削除ボタンを押し、確認ダイアログで「削除」を選択する
- **THEN** 当該エントリは永続化ファイルとホーム画面の両方から取り除かれる

#### Scenario: 削除を取り消す

- **WHEN** ユーザーが削除ボタンを押し、確認ダイアログで「キャンセル」を選択する
- **THEN** エントリは削除されず、何の変更も発生しない

### Requirement: アイコン画像の取り込みと保存

システムは、ユーザーが選択した画像ファイル（PNG / JPEG）を取り込み、アプリのサポートディレクトリへコピー保存し、エントリと紐付ける SHALL。

#### Scenario: アイコンを選択して登録する

- **WHEN** ユーザーがファイル選択ダイアログから画像を選び確定する
- **THEN** システムは画像をアプリのサポートディレクトリ配下にコピーし、エントリにそのパスを保存する

#### Scenario: アイコンを未指定で保存する

- **WHEN** ユーザーがアイコンを指定せずにエントリを保存する
- **THEN** システムはデフォルトアイコンを当該エントリへ割り当てる
