## ADDED Requirements

### Requirement: 指定ディレクトリでの PTY 上 `claude` 起動

システムは、与えられたランチャーエントリのリポジトリ絶対パスを作業ディレクトリとして、PTY（擬似端末）上で `claude` CLI を起動し、指定 Skill を実行するための入力を渡す SHALL。

#### Scenario: 正常に起動する

- **WHEN** 実行画面が開かれ、対応エントリのリポジトリと Skill 名が引き渡される
- **THEN** システムは PTY をそのディレクトリで生成して `claude` を実行し、Skill 起動コマンドを PTY の入力ストリームへ送る

#### Scenario: `claude` バイナリが見つからない

- **WHEN** `claude` CLI が PATH に存在せず、PTY 起動が失敗する
- **THEN** システムは実行画面に「`claude` コマンドが見つかりません。インストールと PATH を確認してください」とエラーを表示し、再試行ボタンを提供する

#### Scenario: リポジトリディレクトリが消失している

- **WHEN** エントリに保存されたリポジトリパスがアプリ起動後に削除されており、ディレクトリが存在しない
- **THEN** システムは PTY を生成せず、「リポジトリディレクトリが見つかりません」とエラーを表示する

### Requirement: PTY 入出力ストリームとサイズ制御の公開

システムは、PTY の出力（標準出力 / 標準エラーが混合した端末出力）をバイト列ストリームとして購読可能にし、PTY への書き込み Sink と端末サイズ変更 API を提供する SHALL。

#### Scenario: 出力ストリームを購読する

- **WHEN** `embedded-terminal` が `skill-runner` の出力ストリームを購読する
- **THEN** PTY 出力のバイト列が時系列順にイベントとして配信される

#### Scenario: 入力を書き込む

- **WHEN** `embedded-terminal` がユーザーのキー入力バイト列を `skill-runner` の入力 Sink に書き込む
- **THEN** 当該バイト列が即座に PTY へ送信され、`claude` プロセスに届く

#### Scenario: 矢印キーや Ctrl 修飾キーを送信する

- **WHEN** ユーザーがターミナル上で矢印キー / Ctrl-C / Ctrl-D を入力する
- **THEN** 対応する制御シーケンスがそのまま PTY へ送られる

#### Scenario: 端末サイズの変更を伝える

- **WHEN** `embedded-terminal` が端末の cols / rows を更新する
- **THEN** システムは PTY に対し対応するサイズ変更（`pty.resize`）を呼び出す

### Requirement: 対話的入力のサポート

システムは、`claude` 側の承認プロンプト（y/n）・選択 UI（矢印キー）・パスワード入力など、**TTY 前提の対話的入力を一通りサポートする** SHALL。本 change は当該シナリオを「動作する」前提として扱い、`Process.start` ベースの劣化動作にフォールバックしない。

#### Scenario: 承認プロンプトに応答する

- **WHEN** `claude` が `(y/n)` のプロンプトを表示し、ユーザーが `y` + Enter を入力する
- **THEN** PTY に `y\n` が送られ、`claude` は承認として扱う

#### Scenario: 矢印キーで選択 UI を操作する

- **WHEN** `claude` が矢印キー操作の選択 UI を表示し、ユーザーが上下矢印を入力する
- **THEN** 対応する ANSI 制御シーケンスが PTY に送られ、UI のハイライトが移動する

### Requirement: プロセスのライフサイクル管理

システムは、PTY 上の子プロセスの開始・終了・キャンセルを観測可能な状態として公開し、ユーザーがキャンセルできるようにする SHALL。

#### Scenario: プロセスが正常終了する

- **WHEN** PTY 上のプロセスが終了コードを返して終了する
- **THEN** システムは状態を「完了（終了コード N）」に遷移させ、ターミナル UI に終了コードを表示する

#### Scenario: ユーザーがキャンセルする

- **WHEN** ユーザーが実行画面のキャンセルボタンを押す
- **THEN** システムは PTY 経由で SIGTERM を送り、状態を「キャンセル済み」に遷移させる

#### Scenario: 実行画面を離脱する

- **WHEN** ユーザーが実行中に画面外へ遷移する
- **THEN** システムは PTY を破棄し、子プロセスを終了させ、リソースを解放する
