## MODIFIED Requirements

### Requirement: トップバー常駐のアクティビティモニタ

アプリ SHALL トップバーのアクション領域に、メモパッドアイコンと設定アイコンの
左へアクティビティモニタを常設表示する。モニタは **CPU・メモリ・ディスク I/O・
ネットワーク I/O** の 4 項目からなり、各項目を「アイコン＋ミニレベルバー＋現在
値」で表す。

#### Scenario: モニタがメモパッド・設定の左に並ぶ

- **WHEN** ワークスペース画面が表示されている
- **THEN** トップバー右側に、左から「CPU」「メモリ」「ディスク I/O」「ネット
  ワーク I/O」「メモパッドアイコン」「設定アイコン」の順で並ぶ

#### Scenario: CPU / メモリのミニバーと数値がシステム負荷を反映する

- **WHEN** システム全体の CPU 使用率が x%、メモリ使用率が y% である
- **THEN** CPU モニタはレベルバーを占有率 x% で描画し、横に x%（整数）を表示
  する。メモリ モニタも同様に占有率 y% のバーと y% の数値を表示する

#### Scenario: ディスク I/O のレートが対数バーと人間可読単位で表示される

- **WHEN** システム全体のディスク I/O レート（read + write の合計）が r バイト
  秒である
- **THEN** ディスクモニタはレベルバーを `log10(r)` を `1 KB/s → 1 GB/s` に
  マップした占有率で描画し、横に人間可読単位（`B/s` / `KB/s` / `MB/s` /
  `GB/s`）の数値を表示する。`r < 1 KB/s` はバー 0%、`r > 1 GB/s` はバー 100%
  にクランプする。

#### Scenario: ネットワーク I/O のレートが対数バーと人間可読単位で表示される

- **WHEN** システム全体のネットワーク I/O レート（in + out の合計、ループバック
  除外）が r バイト/秒である
- **THEN** ネットワークモニタはディスクと同じ対数スケールでバーを描画し、横に
  人間可読単位の数値を表示する

#### Scenario: メトリクスが定期的に更新される

- **WHEN** アプリが起動しており時間が経過する
- **THEN** 4 モニタすべてのレベルバーと数値表示は一定間隔（1 秒）で最新の
  システムメトリクスに更新される

### Requirement: システムメトリクスのネイティブ取得

アプリ SHALL macOS ネイティブ層から CPU 使用率（0–100%）・メモリ使用量・メモリ
総容量・**ディスク I/O 累積バイト（read / write）**・**ネットワーク I/O 累積
バイト（in / out）** を取得する。取得は Flutter 標準の `MethodChannel` を介して
行う。

#### Scenario: システムメトリクスを取得する

- **WHEN** ViewModel がポーリング周期でメトリクスを要求する
- **THEN** ネイティブ層は CPU 使用率・メモリ使用量・メモリ総容量・ディスク
  累積バイト（read / write）・ネットワーク累積バイト（in / out）を返し、
  ViewModel はそれを状態として保持する

#### Scenario: メトリクス取得に失敗する

- **WHEN** ネイティブ層からのメトリクス取得が失敗する
- **THEN** アプリはクラッシュせず、モニタは直近の有効値またはゼロ相当の
  プレースホルダ表示を維持し、次回ポーリングで回復を試みる

#### Scenario: ディスク / ネットワークのレートを差分から計算する

- **WHEN** ViewModel が n 回目のシステムメトリクスを受け取る（n > 1）
- **THEN** ViewModel は前回（n-1 回目）の累積カウンタとの差分を経過秒数で除し、
  ディスク・ネットワークの per-second レートを state に持つ
- **AND** 初回（n = 1）は前回値が無いため、ディスク・ネットワークのレートは
  0 とする

### Requirement: 上位プロセス一覧ポップオーバー

アプリ SHALL CPU / メモリ / ディスク / ネットワークの各モニタのクリックに
応じて、システム上位プロセスの一覧をポップオーバーで表示する。一覧の各行は
プロセス名と、選択した指標に対応する数値を含む（CPU: CPU% + メモリ MB、
メモリ: CPU% + メモリ MB、ディスク / ネットワーク: 該当 I/O のレート）。

#### Scenario: CPU モニタのクリックで CPU 降順の一覧を開く

- **WHEN** ユーザーが CPU モニタをクリックする
- **THEN** ポップオーバーが開き、上位プロセスが CPU 使用率の降順で並び、各行
  にプロセス名・CPU%・メモリ MB が表示される

#### Scenario: メモリ モニタのクリックでメモリ降順の一覧を開く

- **WHEN** ユーザーがメモリ モニタをクリックする
- **THEN** ポップオーバーが開き、上位プロセスがメモリ使用量の降順で並び、各行
  にプロセス名・CPU%・メモリ MB が表示される

#### Scenario: ディスク モニタのクリックでディスク I/O 降順の一覧を開く

- **WHEN** ユーザーがディスクモニタをクリックする
- **THEN** ポップオーバーが開き、上位プロセスがディスク I/O レート（read +
  write）の降順で並び、各行にプロセス名と I/O レート（人間可読単位）が表示
  される
- **AND** ディスク I/O が 0 のプロセスは一覧から除外される

#### Scenario: ネットワーク モニタのクリックでネットワーク I/O 降順の一覧を開く

- **WHEN** ユーザーがネットワークモニタをクリックする
- **THEN** ポップオーバーが開き、上位プロセスがネットワーク I/O レート（in +
  out）の降順で並び、各行にプロセス名と I/O レート（人間可読単位）が表示
  される
- **AND** ネットワーク I/O が 0 のプロセスは一覧から除外される

#### Scenario: プロセス一覧は開いた時点で取得される

- **WHEN** ポップオーバーが開かれる
- **THEN** その時点のプロセス一覧がネイティブ層から取得され、表示される
- **AND** ディスク・ネットワークの一覧取得は最大 1 秒程度の待機時間を伴って
  よく、その間はパネル枠を空表示で出す（スピナーは出さない）

### Requirement: ポップオーバーの開閉挙動

アプリ SHALL ポップオーバーの開閉を直感的に扱えるようにする。4 種類（CPU /
メモリ / ディスク / ネットワーク）のポップオーバーは同時に 1 つしか開かない。

#### Scenario: 外側クリックで閉じる

- **WHEN** ポップオーバーが開いている状態でユーザーがポップオーバーの外側を
  クリックする
- **THEN** ポップオーバーは閉じる

#### Scenario: 同じモニタの再クリックで閉じる

- **WHEN** あるモニタのポップオーバーが開いている状態で、同じモニタを再クリック
  する
- **THEN** そのポップオーバーは閉じる

#### Scenario: 別のモニタへ切り替える

- **WHEN** あるモニタのポップオーバーが開いている状態で、別のモニタをクリック
  する
- **THEN** 元のポップオーバーは閉じ、新たに選んだモニタのポップオーバーが開く
