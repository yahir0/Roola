## ADDED Requirements

### Requirement: Explorer タブ内に常設するファイルプレビューパネル

アプリ SHALL Explorer タブ body を **左ペイン（ディレクトリ一覧）+ 右ペイン
（ファイルプレビュー）** の横分割で構成し、右ペインに読み取り専用の
ファイルプレビューパネルを表示する。プレビューパネルの **表示 / 非表示**
と **split 比率** は per-tab の in-memory 状態として保持し、永続化しない。

#### Scenario: プレビューパネルが Explorer タブの右側に常設される

- **WHEN** Explorer タブを開き、プレビューパネルの可視トグルが ON である
- **THEN** タブ body はディレクトリ一覧（左）とプレビューパネル（右）に
  横分割され、両者の境界は Polaris の split パターンに沿った Drag 可能な
  スプリッタで区切られる

#### Scenario: トグルでプレビューパネルを畳む

- **WHEN** ユーザーが pane header のプレビュートグルを OFF にする
- **THEN** プレビューパネルは非表示になり、ディレクトリ一覧がタブ body の
  全幅を占める

#### Scenario: 表示状態と split 比率はタブごとに独立する

- **WHEN** Explorer タブ A でプレビューを OFF にしてからタブ B に切り替える
- **THEN** タブ B のプレビューパネルの表示状態は影響を受けず、タブ B 自身の
  状態（既定では ON / split 比率 0.6）を維持する

#### Scenario: アプリ再起動でプレビュー設定は既定に戻る

- **WHEN** アプリを終了し再起動する
- **THEN** すべての Explorer タブのプレビューパネル表示状態と split 比率は
  既定値（表示 ON、ratio 0.6）に戻る

### Requirement: 主選択ファイルのシンタックスハイライト表示

アプリ SHALL Explorer タブの主選択（`ExplorerSelection.primary`）が
**テキスト系ファイル** のとき、その内容を `flutter_highlight` 経由で
**シンタックスハイライト付き** で表示する。文字選択 / コピーは可能だが、
編集は不可とする。

#### Scenario: ファイル選択でプレビューが即時表示される

- **WHEN** Explorer の一覧でテキストファイル（例: `.dart`）をクリックして
  主選択にする
- **THEN** プレビューパネルはそのファイルを読み込み、対応する言語の
  シンタックスハイライトを Polaris 配色で描画する

#### Scenario: ディレクトリを選択するとプレビューは空状態になる

- **WHEN** 主選択がディレクトリである、または選択が空である
- **THEN** プレビューパネルは「ファイルを選択してプレビュー」相当の
  空状態 placeholder を表示する

#### Scenario: テキスト内容を選択してコピーできる

- **WHEN** プレビュー表示中のテキストをドラッグまたは ⌘A で範囲選択し ⌘C を押す
- **THEN** 選択範囲のテキストがクリップボードにコピーされる

#### Scenario: プレビューは編集できない

- **WHEN** プレビュー表示領域でキー入力や貼り付けを試みる
- **THEN** ファイル内容は一切変更されず、プレビューは読み取り専用のまま
  維持される

### Requirement: 言語判定は拡張子 → shebang → プレーンテキスト

アプリ SHALL プレビュー対象ファイルの言語を以下の順で判定する:

1. ファイル拡張子から `flutter_highlight` の言語識別子へのマッピング
2. ヒットしなければファイル先頭の shebang（`#!/...`）から判定
3. それでもヒットしなければ言語識別子を持たず、プレーンテキストとして表示

#### Scenario: 拡張子から言語を判定する

- **WHEN** `.dart` ファイルが主選択になる
- **THEN** プレビューは `dart` 言語のシンタックスハイライトで描画される

#### Scenario: shebang から言語を判定する

- **WHEN** 拡張子がなく、ファイル先頭が `#!/bin/bash` で始まるファイルが
  主選択になる
- **THEN** プレビューは `bash` 言語のシンタックスハイライトで描画される

#### Scenario: 未知の拡張子はプレーンテキスト

- **WHEN** マッピングに無い拡張子で shebang も持たないテキストファイルが
  主選択になる
- **THEN** プレビューは色付けなしのプレーンテキストとして描画される

### Requirement: プレビュー対象外ファイルの placeholder

アプリ SHALL バイナリ / 大きすぎ / 読み込みエラーのファイルに対し、
それぞれ専用の placeholder を表示する。アプリはこれらのケースで
クラッシュしない。

#### Scenario: バイナリファイルは「プレビュー不可」を表示

- **WHEN** 先頭 4 KiB に NUL バイトを含む、または UTF-8 として decode 失敗
  するファイルが主選択になる
- **THEN** プレビューは「プレビュー不可（バイナリファイル）」相当の
  placeholder を表示する

#### Scenario: 大きすぎファイルは読み込みを拒否する

- **WHEN** 16 MiB を超えるファイルが主選択になる
- **THEN** プレビューは「ファイルが大きすぎます」相当の placeholder を
  表示し、内容を読み込まない

#### Scenario: 1 MiB 超 16 MiB 以下は先頭のみ表示する

- **WHEN** 1 MiB を超え 16 MiB 以下のテキストファイルが主選択になる
- **THEN** プレビューはファイル先頭 1 MiB だけを読み込み描画し、上部に
  「先頭部分のみ表示しています」相当のバナーを表示する

#### Scenario: 読み込みエラー時にエラーメッセージを表示する

- **WHEN** ファイルが削除済み / 読み取り権限がない 等で読み込みに失敗する
- **THEN** プレビューは「プレビューを表示できません」相当のエラー placeholder
  を表示し、アプリは継続動作する

### Requirement: プレビュー手動リフレッシュ

アプリ SHALL プレビューパネルのヘッダにリフレッシュアクションを提供し、
ユーザー操作で現在のプレビューファイルを再読込できる。アプリは FSEvents
監視（ADR-0041）に連動した自動再読込は行わない（本 change の対象外）。

#### Scenario: リフレッシュで再読込する

- **WHEN** プレビュー表示中にユーザーがパネル右上のリフレッシュアイコンを
  クリックする
- **THEN** プレビュー ViewModel は現在の主選択ファイルを再読込し、最新の
  内容で表示を更新する

### Requirement: Polaris に統合されたハイライトテーマ

アプリ SHALL シンタックスハイライト用のテーマを `PolarisTokens`（ADR-0038）
から組み立てた `Map<String, TextStyle>` で構築する。`flutter_highlight`
標準テーマや色のハードコード（`Color(0x...)` 等）は使用しない。

#### Scenario: プレビューの色は Polaris トークン経由で決まる

- **WHEN** プレビューパネルが描画される
- **THEN** 文字色 / コメント色 / キーワード色 / 文字列色 / 数値色等は
  すべて `PolarisTokens.of(context)` から取得した値で決まり、ソース上の
  色リテラル直書きは存在しない

### Requirement: 既存の右クリック → 外部エディタ経路を維持する

アプリ SHALL ADR-0014 / ADR-0016 で確立した「Roola は閲覧 / 操作、vim
（外部エディタ）は編集」の役割分担を保つ。本 change による読み取り専用
プレビューの追加は、既存の右クリックメニューから vim / OS デフォルト
アプリでファイルを開く経路を一切置き換えない。

#### Scenario: 右クリックから vim でファイルを開ける

- **WHEN** Explorer のファイル行で右クリック → vim メニューを選ぶ
- **THEN** これまで通り vim（または設定された外部エディタ）が当該ファイル
  を開く。プレビューパネルの存在は経路を阻害しない
