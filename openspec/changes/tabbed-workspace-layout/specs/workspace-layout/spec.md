## ADDED Requirements

### Requirement: 3 ペインスロット構成

ワークスペースは `topLeft` / `topRight` / `bottom` の 3 つのペインスロットを持つ。各スロットはタブ群を保持し、タブを 1 つ以上持つスロットを「コンテンツあり」、0 個のスロットを「空」とする。コンテンツありのスロットのみ描画対象とする。

#### Scenario: 3 スロットすべてにコンテンツがある

- **WHEN** `topLeft` / `topRight` / `bottom` のすべてがタブを 1 つ以上持つ
- **THEN** 上段に `topLeft` と `topRight` を左右 2 分割で、下段に `bottom` を配置した 3 ペインレイアウトを描画する

#### Scenario: 空スロットは描画されない

- **WHEN** あるスロットのタブ数が 0 になった
- **THEN** そのスロットは描画対象から外れ、残りのスロットでレイアウトを再フローする

### Requirement: スロット崩し再フロー

コンテンツを持つスロット数に応じてレイアウトを自動的に再フローする。

#### Scenario: 下スロットが空になると上 2 ペイン

- **WHEN** `bottom` スロットのタブがすべて閉じられ、`topLeft` と `topRight` にコンテンツが残っている
- **THEN** `topLeft` と `topRight` の左右 2 分割を画面全高に広げて描画する

#### Scenario: 上段の片方が空になると上下 2 ペイン

- **WHEN** `topLeft` または `topRight` の一方が空になり、残った上段スロットと `bottom` にコンテンツがある
- **THEN** 残った上段スロットと `bottom` を上下 2 分割で描画する

#### Scenario: コンテンツが 1 スロットだけになると単一ペイン全画面

- **WHEN** コンテンツを持つスロットが 1 つだけになった
- **THEN** そのスロットを画面全体に単一ペインとして描画し、従来の単一ペイン表示と同じ見た目になる

#### Scenario: 空スロットへのタブ追加で再び分割される

- **WHEN** 空だったスロットに新しいタブが追加された
- **THEN** そのスロットを含めてレイアウトを再フローし、分割表示に戻す

### Requirement: スプリッタによるリサイズ

レイアウトには上下分割と上段左右分割の 2 本のスプリッタがあり、ユーザーはドラッグでペインの大きさを調整できる。

#### Scenario: 上下スプリッタのドラッグ

- **WHEN** ユーザーが上下スプリッタを上下にドラッグする
- **THEN** 上段と下段の高さ比率が変化し、ドラッグ位置に追従する

#### Scenario: 左右スプリッタのドラッグ

- **WHEN** ユーザーが上段の左右スプリッタを左右にドラッグする
- **THEN** `topLeft` と `topRight` の幅比率が変化し、ドラッグ位置に追従する

#### Scenario: スプリッタは単一ペイン時に表示されない

- **WHEN** レイアウトが単一ペイン全画面の状態である
- **THEN** スプリッタは表示されない

### Requirement: 初回起動時の既定 3 ペイン

永続化されたワークスペースが存在しない、または読み込めない場合、既定の 3 ペインレイアウトを生成する。

#### Scenario: 永続データ無しでの初回起動

- **WHEN** `workspace.json` が存在しない状態でアプリを起動する
- **THEN** `topLeft` にエクスプローラタブ（$HOME）、`topRight` にエクスプローラタブ（$HOME）、`bottom` に素のシェルのターミナルタブ（$HOME）を持つ 3 ペインレイアウトを表示する

#### Scenario: 永続データが壊れている場合

- **WHEN** `workspace.json` が存在するが JSON として不正、または全スロットが空である
- **THEN** 既定の 3 ペインレイアウトを生成して表示する

### Requirement: ワークスペースレイアウトの永続化と復元

ワークスペースのレイアウト（スロット構成・タブ・アクティブタブ・スプリッタ比率）を `workspace.json` に永続化し、起動時に復元する。

#### Scenario: レイアウト変更が永続化される

- **WHEN** タブの開閉・アクティブタブの変更・エクスプローラのパス移動・スプリッタ比率の変更が発生する
- **THEN** 変更後のワークスペースレイアウトを `workspace.json` に保存する

#### Scenario: エクスプローラタブのパス復元

- **WHEN** 永続化されたエクスプローラタブを持つ状態でアプリを再起動する
- **THEN** そのエクスプローラタブを保存時のカレントパスで復元する

#### Scenario: 複数ウィンドウからの保存競合

- **WHEN** 複数ウィンドウが同時に `workspace.json` を書き込む
- **THEN** 後勝ち（last-write-wins）で保存し、データ破壊やクラッシュを起こさない
