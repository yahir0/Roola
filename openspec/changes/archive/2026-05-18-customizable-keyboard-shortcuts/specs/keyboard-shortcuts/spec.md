## ADDED Requirements

### Requirement: コマンドレジストリ

システムはアプリの全アクションを安定 ID（`CommandId`）で一元管理し、各コマンドのメタデータ（カテゴリ・日本語ラベル・アイコン・既定キーコンビ）を静的レジストリ（`CommandRegistry`）として保持する SHALL。メニューバー・コンテキストメニュー・設定画面はこのレジストリを単一の参照元とする。

#### Scenario: コマンドのメタデータ参照

- **WHEN** UI がコマンドのラベル・アイコン・既定キーコンビを必要とする
- **THEN** システムは `CommandRegistry` から当該 `CommandId` のメタデータを返す

### Requirement: キーバインドのカスタマイズと永続化

システムはユーザーが各コマンドのキーコンビを任意に変更でき、変更内容を `<appSupport>/keybindings.json` に永続化する SHALL。未設定のコマンドはレジストリの既定キーコンビを使う。

#### Scenario: キーコンビの変更と保存

- **WHEN** ユーザーが設定画面であるコマンドのキーコンビを変更する
- **THEN** システムは変更を `keybindings.json` に保存し、以後その実効キーコンビを使用する

#### Scenario: デフォルトに戻す

- **WHEN** ユーザーがあるコマンドを「デフォルトに戻す」する
- **THEN** システムはそのコマンドのユーザー上書きを削除し、レジストリの既定キーコンビに戻す

#### Scenario: 未知コマンドの読み飛ばし

- **WHEN** `keybindings.json` にアプリが知らない `CommandId` 文字列が含まれる
- **THEN** システムはその項目を読み飛ばし、残りの設定を正常に読み込む

### Requirement: キー重複の検出と保存ブロック

システムはあるコマンドへ既に別コマンドが使用中のキーコンビを割り当てようとしたとき、警告を表示して保存をブロックする SHALL。修飾キーを 1 つも含まないキーコンビも保存不可とする。

#### Scenario: 重複キーの割り当て

- **WHEN** ユーザーが、別コマンドに割り当て済みのキーコンビを設定しようとする
- **THEN** システムは衝突相手のコマンド名を示して警告し、確定（保存）させない

#### Scenario: 修飾キーなしの割り当て

- **WHEN** ユーザーが修飾キーを含まないキーコンビ（単キー）を設定しようとする
- **THEN** システムは警告し、確定（保存）させない

### Requirement: ネイティブメニューバーからのコマンド実行

システムは macOS ネイティブメニューバー（`PlatformMenuBar`）を提供し、メニュー項目の選択またはその key equivalent でコマンドを実行する SHALL。メニュー項目には実効キーコンビを表示する。

#### Scenario: メニュー項目からの実行

- **WHEN** ユーザーがメニューバーの項目を選択する
- **THEN** システムは対応するコマンドを実行する

#### Scenario: ショートカットからの実行

- **WHEN** ユーザーがコマンドの実効キーコンビを押下する
- **THEN** システムは対応するコマンドを実行する

#### Scenario: ターミナルフォーカス時のショートカット

- **WHEN** ターミナル（SwiftTerm ネイティブビュー）にフォーカスがある状態でコマンドのキーコンビを押下する
- **THEN** システムはメニューバーの key equivalent 経由でコマンドを実行する

#### Scenario: カスタムキーのメニュー反映

- **WHEN** ユーザーがあるコマンドのキーコンビを変更する
- **THEN** メニューバーの当該項目のショートカット表示が即座に更新される

### Requirement: コンテキストメニューでのショートカット表示

システムは右クリック（コンテキスト）メニューの各項目について、コマンドに割り当てられた実効キーコンビを項目の右側に表示する SHALL。

#### Scenario: コンテキストメニュー項目のショートカット表示

- **WHEN** ユーザーがエクスプローラのファイル / フォルダ等を右クリックする
- **THEN** 各メニュー項目の右側に、そのコマンドの実効キーコンビが表示される

### Requirement: コマンドのコンテキスト解決

システムはコンテキスト依存コマンド（選択アイテムやフォーカス中タブを対象とするコマンド）を実行するとき、フォーカス中のタブと選択中アイテムを解決して対象を決める SHALL。対象が解決できないときは何もしない。

#### Scenario: 選択アイテムに対する操作

- **WHEN** ユーザーがエクスプローラで 1 アイテムを選択し、選択アイテム対象のコマンドを実行する
- **THEN** システムは選択中のアイテムを対象としてコマンドを実行する

#### Scenario: 対象が無いコマンドの実行

- **WHEN** 選択アイテムが無い、またはフォーカス中タブが対象種別でない状態でコンテキスト依存コマンドを実行する
- **THEN** システムは何もしない（no-op）
