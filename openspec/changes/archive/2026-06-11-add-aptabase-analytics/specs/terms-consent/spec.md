# Spec: terms-consent

## ADDED Requirements

### Requirement: 規約バージョンの管理

利用規約は整数のバージョン番号を持たなければならない（SHALL）。アプリは現行規約バージョンを定数として保持し、ユーザーが同意した規約バージョンをローカルに永続化しなければならない（SHALL）。本 change での改定を第 2 版とし、現行（2026-06-02 発行）を第 1 版とみなす。

#### Scenario: 同意バージョンの永続化

- **WHEN** ユーザーが同意モーダルで「同意して開始」を選択する
- **THEN** 現行規約バージョンが `privacy_settings.json` に保存される
- **THEN** 以後の起動では同意モーダルが表示されない

### Requirement: 起動時の同意モーダル表示

アプリは起動時に同意状態を照合し、未同意または同意済みバージョンが現行規約バージョンより古い場合、メイン UI の操作前に同意モーダルを表示しなければならない（SHALL）。モーダルは Esc キーやスクリムクリックで回避できてはならない（MUST NOT）。

#### Scenario: 新規ユーザーの初回起動

- **WHEN** 同意記録が存在しない状態でアプリを起動する
- **THEN** 同意モーダルが表示され、背後のメイン UI は操作できない

#### Scenario: 既存ユーザーのアップデート後初回起動（macOS を含む）

- **WHEN** 同意記録がない既存ユーザー（macOS でアップデートしたユーザーを含む）がアプリを起動する
- **THEN** 同意モーダルが表示される

#### Scenario: 規約改定後の再同意

- **WHEN** 第 2 版に同意済みのユーザーが、規約が第 3 版に改定されたアプリを起動する
- **THEN** 同意モーダルが再表示される

#### Scenario: 同意済みなら表示しない

- **WHEN** 現行バージョンに同意済みのユーザーがアプリを起動する
- **THEN** 同意モーダルは表示されず、直ちにメイン UI を操作できる

### Requirement: 同意モーダルの構成

同意モーダルは、規約全文（スクロール可能・アセット同梱の日本語正文）、アナリティクス収集の説明、「使用状況の統計を送信する」トグル（既定 ON）、「同意して開始」ボタン、「終了」ボタンを表示しなければならない（SHALL）。

#### Scenario: 規約全文の閲覧

- **WHEN** 同意モーダルが表示されている
- **THEN** ユーザーはネットワーク接続なしで規約全文をスクロールして読める

#### Scenario: アナリティクスを無効にして同意する

- **WHEN** トグルを OFF にして「同意して開始」を選択する
- **THEN** 規約同意は記録されるが、アナリティクスは無効として永続化され、イベントは送信されない

#### Scenario: 同意を拒否する

- **WHEN** 「終了」を選択する
- **THEN** 同意は記録されず、アプリが終了する

### Requirement: 利用規約の改定

利用規約（`docs/terms-of-use.md`）は、(1) 適用対象 OS に macOS を追加し、(2) 第10条を Aptabase による匿名利用統計の収集実態（収集項目・送信先・オプトアウト手段）に合わせて改稿し、(3) 第 2 版である旨と改定日を明記しなければならない（SHALL）。Windows インストーラ同梱の `license.rtf` は改定後の規約と同一内容でなければならない（SHALL）。

#### Scenario: 規約の整合性

- **WHEN** 規約改定が完了する
- **THEN** `docs/terms-of-use.md`・`assets/terms/terms-of-use.md`・`windows/installer/license.rtf` の 3 点が同一内容（第 2 版）であり、アプリ内の規約バージョン定数が 2 になっている
