## ADDED Requirements

### Requirement: ペインごとに独立したエクスプローラ状態

システムは各エクスプローラタブに独立したカレントパス・ナビゲーション履歴・アイテム選択を持たせる SHALL。1 つのエクスプローラタブの操作は他のエクスプローラタブに影響してはならない（MUST NOT）。

#### Scenario: 2 つのエクスプローラタブが別パスを表示

- **WHEN** `topLeft` のエクスプローラタブをディレクトリ A、`topRight` のエクスプローラタブをディレクトリ B に移動する
- **THEN** 両タブはそれぞれ A・B を表示し続け、互いのカレントパスは独立している

#### Scenario: 履歴はタブごとに独立

- **WHEN** あるエクスプローラタブで戻る / 進む操作をする
- **THEN** そのタブの履歴のみが移動し、他のエクスプローラタブの履歴は変化しない

#### Scenario: アイテム選択はタブごとに独立

- **WHEN** あるエクスプローラタブでファイルを選択する
- **THEN** その選択はそのタブのみに反映され、他のタブの選択状態は変化しない

### Requirement: エクスプローラタブのペインヘッダ

システムは各エクスプローラタブのペインヘッダに戻る / 進む / パスバーを配置する SHALL。これらはウィンドウ共通の AppBar からは取り除かれ、タブ単位で操作する。

#### Scenario: ペインヘッダの戻る / 進む

- **WHEN** ユーザーがエクスプローラタブのペインヘッダの戻る / 進むを操作する
- **THEN** そのタブのナビゲーション履歴を 1 つ戻る / 進む

#### Scenario: パスバーでの移動

- **WHEN** ユーザーがエクスプローラタブのペインヘッダのパスバーでパスを指定する
- **THEN** そのタブのみが指定パスへ移動する

#### Scenario: ウィンドウ AppBar に戻る / 進むが無い

- **WHEN** 3 ペインワークスペースが表示されている
- **THEN** ウィンドウ共通の AppBar には戻る / 進むボタンが無く、ナビゲーション操作は各エクスプローラタブのペインヘッダに集約されている

### Requirement: サイドバー操作の遷移先決定

システムはサイドバーの場所 / お気に入りクリック、および「現在のディレクトリを登録」を、最後にフォーカスされたエクスプローラタブを対象に動作させる SHALL。

#### Scenario: 場所 / お気に入りクリックでフォーカスタブが移動

- **WHEN** ユーザーがサイドバーの場所またはお気に入りをクリックする
- **THEN** 最後にフォーカスされたエクスプローラタブがそのパスへ移動する

#### Scenario: エクスプローラタブが存在しない場合

- **WHEN** エクスプローラタブが 1 つも無い状態でサイドバーの場所 / お気に入りをクリックする
- **THEN** 新しいエクスプローラタブを生成してそのパスを表示する

#### Scenario: 「現在のディレクトリを登録」の対象

- **WHEN** ユーザーがサイドバーの「現在のディレクトリを登録」を実行する
- **THEN** 最後にフォーカスされたエクスプローラタブのカレントパスをお気に入りに登録する
