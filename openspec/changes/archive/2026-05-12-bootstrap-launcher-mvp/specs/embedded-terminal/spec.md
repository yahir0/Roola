## ADDED Requirements

### Requirement: アプリ内フルターミナル UI の表示

システムは、実行画面に `xterm` ベースのフルターミナル UI を組み込み、`skill-runner` の PTY 出力を逐次描画する SHALL。ANSI エスケープ・カラー・カーソル移動・画面クリア等の制御シーケンスを正しく解釈する。

#### Scenario: 出力を逐次表示する

- **WHEN** `skill-runner` の出力ストリームからバイト列が届く
- **THEN** ターミナル UI 上にその内容が ANSI 制御を解釈した状態で表示される

#### Scenario: スクロールバック

- **WHEN** 出力が画面分量を超える
- **THEN** ユーザーは過去の出力をマウススクロールで遡って閲覧できる

### Requirement: フルターミナル入力の送信

システムは、ターミナル UI 上のキー入力（修飾キー・矢印キー・ファンクションキー・ペーストを含む）を PTY の入力 Sink に書き込む SHALL。

#### Scenario: テキスト入力を送信する

- **WHEN** ユーザーがターミナル UI に文字を入力し Enter を押す
- **THEN** 入力テキスト + 改行が PTY に書き込まれる

#### Scenario: 承認応答（y / n）

- **WHEN** `claude` 側がプロンプトを表示し、ユーザーが `y` を入力して Enter する
- **THEN** `y\n` が PTY へ送られ、`claude` の承認応答として処理される

#### Scenario: 矢印キーで選択 UI を操作する

- **WHEN** ユーザーが矢印キーを入力する
- **THEN** 対応する ANSI 制御シーケンス（`\x1b[A` 等）が PTY へ送られる

#### Scenario: Ctrl-C による中断

- **WHEN** ユーザーがターミナルにフォーカスしている状態で Ctrl-C を押す
- **THEN** 0x03 が PTY へ送られ、`claude` 側で SIGINT 相当の中断処理が行われる

### Requirement: ウィンドウサイズの追従

システムは、ターミナル UI の表示領域変化に応じて PTY の cols / rows を更新し、リサイズに追従する SHALL。

#### Scenario: ウィンドウをリサイズする

- **WHEN** ユーザーがアプリウィンドウのサイズを変更し、ターミナル UI の表示 cols / rows が変わる
- **THEN** システムは `skill-runner` に新しい cols / rows を伝え、PTY が当該サイズへリサイズされる

### Requirement: 実行状態の可視化

システムは、子プロセスの状態（起動中 / 実行中 / 完了 / 失敗 / キャンセル済み）を実行画面のヘッダー領域に表示する SHALL。

#### Scenario: 状態遷移を表示する

- **WHEN** 子プロセスの状態が変化する
- **THEN** ヘッダー領域のインジケーター（ラベルとカラー）が当該状態へ即時更新される

### Requirement: 終了後の操作

システムは、子プロセス終了後にもターミナル出力を保持し、ユーザーが再実行 / ホームへ戻る操作を選択できるようにする SHALL。

#### Scenario: 終了後の再実行

- **WHEN** 子プロセスが終了した後にユーザーが「再実行」ボタンを押す
- **THEN** 同じエントリで新たな PTY を生成し、`claude` を再度起動する

#### Scenario: ホームへ戻る

- **WHEN** ユーザーが「ホームへ戻る」ボタンを押す
- **THEN** 実行画面が破棄され、ホーム画面へ遷移する（実行中であればキャンセルされる）
