## ADDED Requirements

### Requirement: トップバー常駐のアクティビティモニタ

アプリ SHALL トップバーのアクション領域に、メモパッドアイコンと設定アイコン
の左へアクティビティモニタを常設表示する。モニタは CPU とメモリの 2 項目から
なり、各項目を「アイコン＋ミニレベルバー＋現在値（%）」で表す。

#### Scenario: モニタがメモパッド・設定の左に並ぶ

- **WHEN** ワークスペース画面が表示されている
- **THEN** トップバー右側に、左から「CPU モニタ」「メモリ モニタ」
  「メモパッドアイコン」「設定アイコン」の順で並ぶ

#### Scenario: ミニバーと数値がシステム負荷を反映する

- **WHEN** システム全体の CPU 使用率が x%、メモリ使用率が y% である
- **THEN** CPU モニタはレベルバーを占有率 x% で描画し、横に x%（整数）を
  表示する。メモリ モニタも同様に占有率 y% のバーと y% の数値を表示する

#### Scenario: メトリクスが定期的に更新される

- **WHEN** アプリが起動しており時間が経過する
- **THEN** モニタのレベルバーと数値表示は一定間隔（1 秒）で最新の
  システムメトリクスに更新される

### Requirement: システムメトリクスのネイティブ取得

アプリ SHALL macOS ネイティブ層から CPU 使用率（0–100%）・メモリ使用量・
メモリ総容量を取得する。取得は Flutter 標準の `MethodChannel` を介して行う。

#### Scenario: システムメトリクスを取得する

- **WHEN** ViewModel がポーリング周期でメトリクスを要求する
- **THEN** ネイティブ層は CPU 使用率・メモリ使用量・メモリ総容量を返し、
  ViewModel はそれを状態として保持する

#### Scenario: メトリクス取得に失敗する

- **WHEN** ネイティブ層からのメトリクス取得が失敗する
- **THEN** アプリはクラッシュせず、モニタは直近の有効値またはゼロ相当の
  プレースホルダ表示を維持し、次回ポーリングで回復を試みる

### Requirement: 上位プロセス一覧ポップオーバー

アプリ SHALL CPU モニタまたはメモリ モニタのクリックに応じて、システム上位
プロセスの一覧をポップオーバーで表示する。一覧の各行はプロセス名・CPU
使用率（%）・メモリ使用量（MB）を含む。

#### Scenario: CPU モニタのクリックで CPU 降順の一覧を開く

- **WHEN** ユーザーが CPU モニタをクリックする
- **THEN** ポップオーバーが開き、上位プロセスが CPU 使用率の降順で並び、
  各行にプロセス名・CPU%・メモリ MB が表示される

#### Scenario: メモリ モニタのクリックでメモリ降順の一覧を開く

- **WHEN** ユーザーがメモリ モニタをクリックする
- **THEN** ポップオーバーが開き、上位プロセスがメモリ使用量の降順で並び、
  各行にプロセス名・CPU%・メモリ MB が表示される

#### Scenario: プロセス一覧は開いた時点で取得される

- **WHEN** ポップオーバーが開かれる
- **THEN** その時点のプロセス一覧がネイティブ層から取得され、表示される

### Requirement: ポップオーバーの開閉挙動

アプリ SHALL ポップオーバーの開閉を直感的に扱えるようにする。CPU とメモリの
ポップオーバーは同時に 1 つしか開かない。

#### Scenario: 外側クリックで閉じる

- **WHEN** ポップオーバーが開いている状態でユーザーがポップオーバーの外側を
  クリックする
- **THEN** ポップオーバーは閉じる

#### Scenario: 同じモニタの再クリックで閉じる

- **WHEN** あるモニタのポップオーバーが開いている状態で、同じモニタを
  再度クリックする
- **THEN** ポップオーバーは閉じる

#### Scenario: CPU とメモリのポップオーバーは排他

- **WHEN** CPU のポップオーバーが開いている状態でメモリ モニタをクリックする
- **THEN** CPU のポップオーバーは閉じ、メモリのポップオーバーが開く

### Requirement: Polaris デザインシステム準拠

アクティビティモニタとポップオーバーの全表示要素 SHALL Polaris デザイン
システム（ADR-0038）に準拠する。色・余白・角丸・タイポグラフィは
`PolarisTokens` 経由で参照し、`Color(0x...)` やマジックナンバーを
ハードコードしてはならない（MUST NOT）。

#### Scenario: デザイン値はトークン経由で参照される

- **WHEN** アクティビティモニタおよびポップオーバーが描画される
- **THEN** 使用される色・余白・角丸・フォントは `PolarisTokens` から取得
  された値であり、コンポーネント側にハードコードされた値を含まない

#### Scenario: 値の変化は即時反映される

- **WHEN** メトリクス値が更新される、またはポップオーバーが開閉する
- **THEN** 表示はトゥイーンアニメーションなしで即時に変化する
  （ADR-0038 D7）
