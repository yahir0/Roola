# ADR-0046: Explorer に読み取り専用ファイルプレビューパネルを追加する

- **Status**: Accepted
- **Date**: 2026-05-21

## Context

現状、Explorer タブでファイルを軽く中身確認したいとき、ユーザーは
右クリック → vim（または OS デフォルトアプリ）でファイルを開くしかない。
`.txt` / `.dart` / `.md` などを「1 行確認したい」「依存パッケージ名だけ見たい」
程度の用途には vim や別アプリ起動は重く、操作コストに対するリターンが低い。

一方、Roola は [ADR-0014] / [ADR-0016] で **エクスプローラ + 汎用ランチャー**
として定義しなおしており、フル機能エディタを内包することは製品の核を
ぶれさせる。`flutter_code_editor` のような編集 UI を持つライブラリを入れる
と、「読み取り専用にしたつもりだが編集 UI が内包されている」状態になり、
いずれ機能要望に押されてエディタ化する誘惑を抱える。

Roola に欲しいのは **「ちょっと見る」ための軽量プレビュー** であって
エディタではない。Finder の Column View 右ペインや QuickLook（Space キー）の
発想に近い。

[ADR-0014]: 0014-explorer-first-ui.md
[ADR-0016]: 0016-generalize-launcher-action.md

## Decision

Explorer タブの body を **左ペイン（既存ディレクトリ一覧）+ 右ペイン
（読み取り専用ファイルプレビュー）** の横分割にする。Explorer タブの
主選択（[ExplorerSelection].primary）がファイルのとき、その内容を右ペインに
シンタックスハイライト付きで表示する。

- **編集機能は持たない**。`SelectableText` ベースで選択 / コピーのみ可
- ライブラリは **`flutter_highlight`** のみ採用する（`flutter_code_editor` は
  採用しない）
- テーマは Polaris トークン（[ADR-0038]）からハイライト用 `Map<String, TextStyle>` を組み立てる
- スコープは **テキスト系ファイルのみ**。画像 / PDF / バイナリは非対象
- パネルは tab pane header のトグルで表示 / 非表示を切り替えられる。デフォルトは **表示**
- 表示状態 / 横幅は **per-tab の in-memory 状態**。永続化しない（[ADR-0042]
  と整合：ワークスペースは終了時破棄）

[ExplorerSelection]: ../../lib/ui/explorer/explorer_item_selection.dart
[ADR-0038]: 0038-polaris-design-system.md
[ADR-0042]: 0042-discard-workspace-on-exit.md

### サポートする / しないファイル

| 区分 | 動作 |
| --- | --- |
| テキスト系（UTF-8 として読み出せる） | プレビュー表示 |
| バイナリ（先頭 4 KiB に NUL を含む） | 「プレビュー不可（バイナリ）」表示 |
| 1 MiB 超 | 先頭 1 MiB を切り取って「先頭部分のみ表示」バナー付き |
| 16 MiB 超 | 読み込みを拒否し「ファイルが大きすぎます」表示 |
| 読み込みエラー（権限 / 削除済み） | エラーメッセージ表示 |
| ディレクトリ選択 / 非選択 | 空状態（「ファイルを選択してプレビュー」） |

### シンタックスハイライトの言語判定

1. 拡張子で `flutter_highlight` の言語識別子を引く（`.dart` → `dart` 等）
2. ヒットしなければ shebang を確認（`#!/bin/bash` → `bash` 等）
3. それでもヒットしなければプレーンテキストで表示（色付けなし）

### 更新タイミング

- 主選択が変わったら再読込
- パネル右上の **リフレッシュボタン** で手動再読込
- FSEvents 監視（[ADR-0041]）の自動再読込は **やらない**（プレビュー中だけ
  別フックを足す価値が薄く、ユーザーが必要時に手動更新で足りる）

[ADR-0041]: 0041-realtime-fs-watch.md

### per-tab 状態のかたち

[ADR-0027] に倣い、`filePreviewViewModelProvider.family(tabId)` を作る。
内部で `explorerSelectionProvider(tabId)` を watch し、主選択ファイルを
`AsyncNotifier` で読み出す。エクスプローラ以外のタブ種別（ターミナル / Git）
には作らない。

[ADR-0027]: 0027-per-tab-state-via-family.md

## Why

- **「ちょっと見たい」を vim 経由から外せる**: ファイル選択だけでゼロ
  アクションで中身が見える。Finder の Column View / QuickLook と同じ
  操作モデルで馴染みやすい
- **意図的に天井を低く作る**: `flutter_highlight` は色付けと選択コピー
  だけの最小依存。編集 UI を持ち込まないことで [ADR-0014] / [ADR-0016] の
  製品定義を守る。編集したくなったら vim 右クリックという既存導線が
  そのまま使える
- **Polaris に統合できる**: `flutter_highlight` は `Map<String, TextStyle>`
  で渡すテーマ定義なので、Polaris トークンから組み立てたマップを使えば
  色のハードコードを避けられる（[ADR-0038] D1 準拠）
- **Explorer タブ内分割が一番素直**: ワークスペースに新タブ種別を足す案
  もあるが、「選択 → 即見える」をやるなら同一タブ内に置くのが UX 的に
  正解。スプリッタ 1 本追加で済む

## 代替案

### 代替案 1: `flutter_code_editor` を `readOnly:true` で採用する

- 行番号 / コードフォールディング / 検索など読み取り体験は厚い
- 編集 UI を内包したまま使うことになり、いずれ「ちょっとだけ編集」要望が
  押し寄せたとき断る根拠が弱くなる。エディタ化への誘惑が依存ライブラリの
  存在自体から生じる
- Roola の役割（[ADR-0014] / [ADR-0016]）を守るには、ライブラリ側で天井を
  決めてくれるほうが安全。却下

### 代替案 2: macOS の QuickLook を呼ぶ（`qlmanage` / NSWorkspace）

- アプリが何も持たない最軽量案
- `.dart` 等プログラミング言語のシンタックスハイライトは出ない / 弱い
  （Quick Look は実質テキストとして表示するだけ）
- 見た目を Polaris に揃えられない（OS デフォルトのカードが出る）
- 「Roola 内で完結する」体験が崩れる。却下

### 代替案 3: Explorer タブ内ではなく、ワークスペースに「preview」タブ種別を追加

- 既存の 3 ペイン × タブ構造（[ADR-0026]）にプレビュー専用タブを足す
- 「ファイル選択 → 即プレビュー」を実現するには「どのプレビュータブに
  反映するか」というルーティング問題が発生する。複数 Explorer タブとの
  紐付けが複雑
- Explorer タブ内分割なら 1:1 で完結する。却下

[ADR-0026]: 0026-three-pane-tabbed-workspace.md

### 代替案 4: Space キーでオーバーレイ表示（Finder QuickLook 互換）

- 馴染みのある UX
- 常時表示でないため「選択 → 即見える」のゼロアクションを満たさない
- 後付けで追加する余地は残る（常設パネルと両立可能）。第 1 弾としては
  常設パネル優先で却下

## Trade-offs

- **Explorer タブの最小幅が増える**: 横分割が入るので狭いウィンドウだと
  プレビューを畳む必要が出る。トグルで折りたためばよい
- **読み込みは IO**: ファイル読み出しは `AsyncNotifier` 経由の非同期。
  1 MiB 弱までは UI を阻害しない見込み。大きいファイルは切り捨て表示で
  保護する
- **シンタックスハイライト網羅性は `flutter_highlight` 依存**: 100+ 言語
  に対応するが、未対応言語はプレーンテキスト表示になる。実害が出れば
  ファイル先頭判定（shebang / `<?xml` 等）を厚くする余地あり
- **行番号は持たない**: コード読みに行番号があると便利だが、Polaris 的
  には装飾は最小化したい。要望が出てから後付け検討する
- **FSEvents 連携をスキップ**: ターミナル等で書き換えても自動更新されない。
  リフレッシュボタンで手動更新する運用。実害が出れば [ADR-0041] の
  watcher にフックを足す
