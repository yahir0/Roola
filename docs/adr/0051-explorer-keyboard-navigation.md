# ADR-0051: エクスプローラ一覧を十字キーで操作できるようにする

- **Status**: Accepted
- **Date**: 2026-05-22

## Context

エクスプローラの一覧（[ADR-0026] の 3 ペインタブ式ワークスペース内の
ディレクトリビュー）は、操作モデル上シングルクリックで選択、ダブルクリックで
遷移／オープンとしてきた（[ADR-0021] / [ADR-0038] D12）。選択状態は
`ExplorerItemSelection`（`family(tabId)`）が「主選択 1 件 + 複数選択集合」の
モデルで保持する。

しかしキーボードだけでは一覧のアイテムを選べなかった。マウスでクリックする
以外に選択を動かす手段がなく、十字キー（矢印キー）でフォルダ／ファイルを
辿れないという指摘があった。デスクトップのファイラとしては、上下キーで選択を
動かし、Enter で開く、という操作が期待される。

一方、キー割り当ては [ADR-0033] のコマンドレジストリ + ネイティブメニュー
バーで統一しているが、矢印キーはメニューの key equivalent に載せると
**テキストフィールド（パスバー・各種ダイアログ）入力中まで奪ってしまう**。
矢印キーによる一覧内ナビゲーションは「その一覧にフォーカスがあるときだけ」
効くべきローカル操作であり、グローバルコマンドには適さない。

## Decision

一覧のキーボード操作を **`Focus` ベースのローカルハンドラ**として実装する。
コマンドレジストリ（[ADR-0033]）には載せない。

- `_DirectoryListing` の `CustomScrollView` を `Focus` で包み、`onKeyEvent`
  で矢印キー / Enter を処理する。`Listener` の `onPointerDown` で一覧内の
  ポインタ押下時にこの `Focus` へフォーカスを移すため、タイルのタップ選択
  （`GestureDetector`）と両立する（`Listener` はジェスチャーアリーナと独立に
  pointer down を受ける）。
- キーマップ:
  - `↑` / `↓`: 主選択を上下に 1 件動かす（単一選択。未選択なら端から開始）。
    キーリピートにも追従する。
  - `Enter` / `NumpadEnter`: 主選択を開く（ディレクトリは `navigateTo`、
    ファイルは `FileOpener` で OS デフォルトアプリ）。
  - `→`: 主選択がディレクトリならそこへ侵入する（ファイルでは何もしない）。
  - `←`: 親ディレクトリへ戻る（`goUp`）。
- 選択行が画面外に出たら、最小限だけ `ScrollController.jumpTo` でビューポート
  内に送る。行高は密度設定（[ADR-0024]）と Skill サブタイトル有無で可変なため、
  先頭からの累積高を `explorerRowHeight` と同じ条件で算出して正確に求める。
- 処理したキーのみ `KeyEventResult.handled` を返し、それ以外は `ignored` で
  既存のショートカット機構へ通す。テキストフィールド入力中はこの `Focus` が
  フォーカスを持たないため、矢印キーは横取りされない。

## Consequences

- マウスを使わずにフォルダ／ファイルを選択・オープンできるようになる。一覧を
  一度クリック（またはアイテム選択）してフォーカスを得れば、以降は十字キーで
  辿れる。
- 選択モデルは既存の `ExplorerItemSelection`（主選択）をそのまま使うため、
  コンテキストメニュー・コマンド（[ADR-0033]）の「選択中アイテム対象」処理と
  自然に整合する。
- 複数選択（Shift+矢印でのレンジ選択）は対象外。必要になれば別途拡張する。
- 矢印キーをグローバルコマンド化しないため、入力フィールドとの競合は起きない。
