# Design System (Polaris)

Roola の独自デザインシステム **Polaris** の現行規約を定義する。これは**生きた
リファレンス**であり、Polaris の現在の姿を知りたいときはここを読む。判断の背景・
反復の経緯は `docs/adr/` を参照する。

## 位置付けと規範ソース

- **思想（不変）**: 機能主義（functionalism）。装飾を排し、機能・精密さ・情報
  密度で形を決める。ディーター・ラムス → ジョナサン・アイブの系譜（「より少なく、
  しかしより良く」）。
- **ビジュアル（導出物・可変）**: 色・寸法・タイポグラフィは思想からの導出であり、
  機能的に優れる選択があれば差し替えてよい。
- 体系名 **Polaris** = アプリ名 Roola（移動呪文ルーラ）に対する「移動のための導き
  の星」。不動の原理から可変のデザインが導かれる構造を一語で表す。

判断の時系列ログは ADR にある。本ドキュメントはその現行スナップショットで、ADR が
更新されたらここも更新する。

- ADR-0038: Polaris デザインシステムを採用する（ADR-0020 を Supersede）— 基盤の D1〜D14
- ADR-0054: コンテンツ面はベゼル付きディスプレイに統一し、内側はフラットにする — D1/D3/D6 の一般化

実装の単一ソースは `lib/app/theme.dart` の `PolarisTokens`（`ThemeExtension`）。
本ドキュメントの値とコードが食い違ったら**コードが正**。

## デザイントークン（一元管理）

Polaris の全デザイン値は `PolarisTokens` という単一の `ThemeExtension` に集約する。

- UI コンポーネントは `Theme.of(context)` 経由で `PolarisTokens` を参照する（MUST）。
- リテラルの `Color(0x...)` やマジックナンバー（角丸・余白）をコンポーネント側に
  ハードコードしない（MUST NOT）。
- 余白は `PolarisTokens` の `static const`（`space1`=4px 〜）を使い、`const`
  構築の中でもリテラルを書かない。

主なトークン（`lib/app/theme.dart`）:

| トークン | 役割 |
|---|---|
| `machine` | 機材本体の地（ターミナル）。`bg` 直下までリフトしたクールグラファイト |
| `well` | 沈んだ計器ディスプレイの地（ファイル一覧・コンテンツ面）。クールグラファイト |
| `bg` | 筐体の枠（トップバー / サイドバー / ステータスバー）。`well` より一段明るい |
| `topEdge` | 筐体上端が光を受ける 1px ハイライト |
| `surface` / `surfaceHi` | ホバー行の塗り / 選択行の控えめな塗り |
| `line` | パネルの継ぎ目（1px ライン） |
| `text` / `textDim` / `textFaint` | 主 / 副 / 最弱トーンのテキスト |
| `accent` / `onAccent` | 限定使用のアクセント（1 色）/ その上のテキスト・アイコン |
| `signalNew` / `signalModified` / `signalConflict` | Git 信号色（新規 / 変更 / コンフリクト） |
| `radius` | 機械加工 R（角丸 px、既定 4px） |
| `gridUnit` / `space1..` | 4px グリッド単位と余白スケール |

## カラーとトーン階層

### ダーク専用

ダーク専用の Polaris テーマのみを提供する（MUST）。ライトテーマを提供しない
（MUST NOT）。ライト / ダークの切替手段は存在しない。「沈んだ計器ディスプレイ」
「暗い地のかっこよさ」はライトでは別物になるため。

### グラファイトの 3 トーン階層

地は純黒でなくグラファイト（僅かにリフトした暗色）。面は機能で 3 トーンに分ける:

- `well` — **計器を読む面**（ファイル一覧・コンテンツ面）。最も沈み込む。
- `machine` — **機械を叩く面**（ターミナル）。`bg` 直下までリフトし、操作面は
  シャシ表面と同じ高さという機材メタファに合わせる。`well` と同じクール系を維持し、
  差別化は**深さでなくリフト幅**で行う。
- `bg` — **筐体の枠**（トップバー / サイドバー / ステータスバー）。一段明るい。

面の分離に drop shadow / elevation を使わない（MUST NOT）。区切りはトーン差と
1px ライン（`line`）、および筐体上端の `topEdge` ハイライトのみで表現する。

### 単一アクセント（ゴールド / アイスブルー切替）

- アクセントは常に **1 色のみ**運用する（MUST）。同時に複数のアクセントを併用
  しない（MUST NOT）。
- その 1 色をユーザー設定で **暖色ゴールド `#D0A341`（既定）** と **寒色アイス
  シアン `#48C9DE`** から選べる。切り替えても常に 1 色で、混在しない。
- 設定は `AppearanceSettings` に `PolarisAccent` として永続化し、
  `AppTheme.polaris(accent:)` がトークンの `accent` を差し替える。
- アクセントは**選択・現在地・フォルダ・アクティブな操作対象**など機能的に意味の
  ある箇所に限定使用する。それ以外は無彩色のグラファイトのまま保つ。

### Git 状態の信号色

ファイルの Git 作業ツリー状態は意味専用の信号色で示す（MUST）。新規 / 変更 /
コンフリクトを区別できる色で表す。「変更」(`signalModified`) はアクセントのゴールド
と色相で明確に区別できる色（steel blue 系）でなければならない（MUST）。ファイル種別
ごとの色分けは行わない（MUST NOT）。状態のないファイルにはマーカーを出さない。

## 形状・レイアウト

### 機械加工 R = 4px と 4px グリッド

- パネル・カード・ダイアログ・メニュー等の角は 4px の角丸に統一する（MUST）。
  値は `PolarisTokens.radius` の単一トークンで統制し、ハードコードしない（MUST NOT）。
- 0px のナイフエッジや大きな角丸は使わない。
- レイアウトの全寸法（行高・バー高・余白）は 4px の倍数に乗せる（MUST）。4px グリッド
  の厳密さそのものが機械の質感になる。

### コンテンツ面はベゼル付きディスプレイに統一（ADR-0054）

画面の本文エリア全体を、単一の `PolarisDisplayPanel`（`well` トーン・8px インセット・
R=4px・1px ベゼル）に嵌める。これは「計器ディスプレイパネル」を Explorer のファイル
一覧だけでなく**コンテンツ面一般の不変ルール**へ一般化したもの。

- ベゼルの単位は **画面につき 1 枚**。セクションごとに箱を作らない（Material 定番形
  または「重い箱」に読め、機能的意味も持たないため）。
- **ベゼルの内側はフラット**にする。面の塗りや枠で要素を囲わず、余白と 1px ヘアライン
  だけで構成する。「読む面＝沈み込み、操作する面＝フラット」という機能差に従う
  （preference フォームは操作する面なので内側を階層化しない）。
- ターミナルだけは例外で、計器パネル（well + インセット）でなく、ヘッダ（bg）と
  機材面（machine）を 1px 継ぎ目で分けるフラットなフルブリード構成にする。

設定画面まわりの実装部品: `lib/ui/common/polaris_settings_panel.dart`
（`PolarisSettingsSection` / `PolarisSectionDivider` / `PolarisFieldLabel`）。

### secondary 画面はモーダルシェル（ADR-0054 D5）

設定 / ランチャー管理 / キーバインド / エントリ編集は「ナビで深く潜った先」でなく
「ワークスペースに重ねる一時的なモーダル文脈」として扱う。

- `PolarisModalShell`（`lib/ui/common/polaris_modal_shell.dart`）で統一する。
- ルートは `opaque: false` で push し、背後にワークスペースを描く。スクリム
  （背後を薄暗く沈める）＋中央のベゼル付きディスプレイとして出す。
- 閉じる手段は **✕ / スクリムタップ / Esc** の 3 つ。`←戻る` 山括弧 + タイトル
  ヘッダはこれらの画面から撤去する（メイン `WorkspacePage` と license 系は従来の
  `MacosWindowAppBar` を引き続き使用）。

### 既存レイアウトの維持

Polaris の適用は**視覚層のスタイル差し替えに限る**。どの画面が存在するか、タブ /
ペイン / サイドバーの配置、ナビゲーション構造、各画面内のウィジェット配置順を
変更しない（MUST NOT）。Polaris は視覚層の方針であって、レイアウト仕様ではない
（macOS タイトルバー統合はウィンドウ枠の扱いで、ここでいう既存レイアウトに含まない）。

## モーション

UI のフィードバックは即時（0ms）でなければならない（MUST）。次を使わない（MUST NOT）:

- ink splash（`splashFactory: NoSplash`）
- ホバーのフェード（状態変化は次フレームで即座に）
- ページ遷移アニメーション（`PageTransitionsBuilder` で即時化）
- スクロールの慣性 / バウンス / オーバースクロールグロー（`PolarisScrollBehavior`
  + `ClampingScrollPhysics`）

フェードやバウンスは「Flutter らしい柔らかさ＝モバイル感」の発生源のため排除する。

## タイポグラフィ

- クローム（ラベル・ファイル名・本文）はプロポーショナルフォント（SF Pro）。
- 桁を揃えるデータ（パーミッション・サイズ・日時・パス）は等幅フォント（SF Mono、
  フォールバック Menlo）で描画し、各行で桁を揃える（MUST）。
- 日本語は Hiragino Sans へフォールバックして文字化け・欠落なく表示する（MUST）。
- 本文・ラベルのウェイトは **w500 基準**、ロゴワードマークのみ w600。精密さは
  ウェイトでなくサイズ・トラッキング・トーン階層で出す。
- 全面等幅は「ターミナル / コードエディタの安っぽい字面」になるため使わない。

## アイコン・インジケーター

### モノラインアイコン（自作描画）

フォルダ・ファイル等の型アイコン、および UI 各所のグリフは、幾何学的なモノライン
（ヘアラインのストローク）として `CustomPaint` で自前描画する（MUST）。既製の
アイコンフォント / アイコンセット（Material / Cupertino の `Icons.*`）に依存しない
（MUST NOT）。共通グリフは `PolarisGlyph`（`lib/ui/common/polaris_glyphs.dart`）。

### 表示灯型の状態インジケーター

状態を示す小さな点状インジケーター（Git 状態・ステータス）は、単色の塗り点でなく、
ヘアラインのリング（淡）＋塗りのコア（濃）からなる同心円＝「ベゼルに嵌った表示灯」
として描画する（MUST）。

### 切替 UI・スライダー

- 切替は M3 `SegmentedButton` を使わず `PolarisToggle`（`lib/ui/common/polaris_toggle.dart`）
  を使う。外枠 1px のみ・仕切り線なし・R=4px、選択は低透過（16%）の淡いアクセント面
  ＋アクセント色の文字で静かに示す。
- スライダーのツマミは Material 既定の丸ツマミでなく、機械加工 R の矩形フェーダ
  キャップ（`theme.dart` の `_FaderThumbShape`）。

## 選択モデル

- ファイル一覧は複数選択に対応する（MUST）。
- 選択行は控えめな塗り（`surfaceHi`）で示す。**主選択（アンカー＝最後に選んだ 1 行）
  のみ**をフルアクセント色で点灯させる（行名・アイコン・左 2px バー）（MUST）。
- キーボードフォーカスは選択と一体で扱い、選択とは別のフォーカス表示を設けない。
- 全選択行をフルアクセントにすると信号性が薄れるため、主選択 1 行に限定する。

## ウィンドウ

- macOS 標準タイトルバーを非表示にし（`window_manager` の `TitleBarStyle.hidden`）、
  アプリ自前のトップバーをタイトルバー兼ドラッグ領域（`DragToMoveArea`）として機能
  させる（MUST）。
- 左端は信号機ボタンの領域（約 78px）を確保する。アプリ全体を「1 個の機械」に見せる。

## 背景の外観モード

アプリ背景の外観モードは次の **2 種類のみ**を提供する（MUST）。任意の単色・画像・
グラデーションへ差し替えるモードは提供しない（MUST NOT）— グラファイト筐体の視覚的
一貫性を保つため（旧 `solid` / `image` / `gradient` は廃止、旧設定値は `opaque` へ移行）。

- `opaque` — 不透明なグラファイト筐体。
- `transparent` — 筐体ごと半透明合成し、背後のデスクトップを透かす。

外観モードに依らず、UI が自前で背景を塗らない領域（ターミナル・設定画面など）でも
不透明なグラファイトの基底層を描画し、透過ウィンドウ素通しの穴を生じさせない（MUST）。
透過は単一の `Opacity` で UI ツリー全体を一括合成して実現する（面ごとに半透明化すると
トーン階層が重なる箇所でアルファが二重掛けされ濁るため）。

## 装飾モチーフは足さない

「機械っぽさ」を出すための applied な装飾（四隅のレジストレーションマーク、選択の照準
ブラケット、特徴的書体の引用等）は使わない（MUST NOT）。機械感は構造・精密さ・抑制
（トーン階層・グリッド・1 アクセント）からのみ滲ませる。**この原則自体が Polaris の核。**
